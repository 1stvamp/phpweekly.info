Articles

Symfony2 components overview: EventDispatcher ♥
ServerGrove's Raul Fraile discusses a favourite design pattern of mine: the mediator, and Symfony2's EventDispatcher is one of the better decoupled implementations. Raul goes through a few examples, as well as discussing using an event subscriber instead of a listener. This is one of those patterns which is gaining traction all across PHP with increased easier use of callbacks and anonymous functions in general.

php7 Interview With Michael Wallner A Full-Time Core PHP Developer – Try To Understand How Things Work & Why They Work That Way ♥ ♥
Khayrattee Wasseem returns with an excellent interview with a PHPc luminary, this time Michael Wallner, who is most recently known for being one of the few developers employed full time by a company (SmugMug) to work on PHP "core". A good interview all round, with lots of solid advice from Michael, and some great insight to what being employed full time to work on an open source project is actually like.

PHP extension writing ♥
Mikko Koppanen, maintainer of many popular extensions such as imagick and php-zmq, shares a note about writing extensions, with a pointer to a sample extension project he has started containing many of the tips and tricks he's gathered, called extsample.

What I took home from ZendCon 2013 ♥
This post from Zend Framework aficionado Michelangelo van Dam covers the amazing time he and many others were treated to this year at ZendCon 2013, from the awesome workshops, to astounding announcements (hellooooo Apigility), high quality talks and of course..the swag. Bless the swag.

Is ORM abstraction a pipe dream? ♥
David Adams gives us this well made write up of his adventures attempting to implement the repository pattern as an easily swappable wrapper around multiple ORMs. Using Doctrine, Propel and Eloquent as the base libraries for his after much wrangling around various APIs and methods, he concludes that while it probably is possible, it's definitely not something that's easily doable as a weekend side project. In the comments Matthieu Napoli makes the good point that the 3 libraries use different ORM patterns, e.g. the Data Mapper pattern, Active Record, and a mixture of the two, which increases the complexity involved in creating an abstraction.

Symfony benchmark using HHVM ♥
Alessandro Tagliapietra published the results from some basic benchmarks running two sample apps (a simple Symfony2 app, and a script to generate fibonachi sequences), using Facebook's HipHopVM (HHVM), PHP 5.4 (+/- APC) and PHP 5.5 (+ OPCache). Some of the results may surprise you, with a simple Symfony app being run much faster by PHP 5.5 with the excellent (formerly Zend) OPCache built in, even over HHVM, but while doing anything the actually uses CPU/memory (okay, it's just a fibonachi generator, but it's something) HHVM comes into it's own, with the JIT kicking in and speeding up certain ops. Read the comments for more discussion of what the results actually mean in real terms.


Projects

ixa-wordpress-core ♥
This is a well maintained and up to date fork of Wordpress with Composer support and available for install from Packagist so you can keep Wordpress maintained like any other dependency in your project, at least until the official patch is decided on and merged.

Fresque ♥
A cool little command line util for managing php-resque workers and queues, and it seems to even allow you to control regular resque workers/queues.

JsonpCallbackValidator ♥
A simple yet amazingly useful idea, this validator checks the content of a JSON-P callback (e.g. a JavaScript callback that will be called with a JSON payload on return to a click) and fails validation if any attempt to run extra code is detected (e.g. an XSS attack).

IcecaveStudios / collections ♥
A collection of..erm..collections, that is collection data types such as lists, queues, maps, sets, etc. based on the C#/.NET collections module, and expanding on the slightly lacking implementations found in the SPL.

3v4l.org ♥
An olde-but-goodie that comes in handy every now and again. Ever want to easily test some code in different versions of PHP? How about over 90 versions? In one go? Via the web? Try it out, but remember not to put anything confidential in those little textareas.
