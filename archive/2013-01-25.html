News & announcements

PHP 5.5.0 Alpha4 released
Latest round of 5.5 alphas is ready to test (I actually missed the 3rd alpha with last weeks lack of an issue). See the news files for alpha3 and alpha4 for the downlow on new features and fixes.

PHP 5.4.11 and PHP 5.3.21 released
Bugfix-a-go-go for your server upgrade delights. Check the changelog for what's fixed.

Laravel 3.2 released
Some bug fixes, but also some great new features such as registration of custom drivers for different systems, e.g. caching, database etc.

Laravel 4 Beta 1 released
L4 is definitely shaping up, with far too any hot new features to list here, of particular note is the new documentation, and the fact that 4 is now the "develop" branch for Laravel, meaning new feature development work after the final release will go into 4.

PSR-3 accepted and ratified
The PHP-FIG have accepted the controversial PSR-3 which defines a standard Logging Interface that logging libraries can conform to if they want, making it easier for frameworks and other libraries to decouple themselves from any particular logging library as long as it implements the PSR3 interface. Love it or hate it, the marmite of the PHP standards world is here to stay.

 

Rasmus to give opening keynote at SunshinePHP
If the awesome line up and activities at the SunshinePHP conference in February weren't enough to entice you to buy a ticket, everyones favourite BDFL will be getting the show on the road, Mr Lerdorf is known for giving great talks so consider adding this event to your calendar.

Audiovisuals

Lately in PHP podcast: The Urge to Kill Older PHP Versions
Regular hosts Manuel Lemos, and making a welcome return after an internet-hiatus Ernani Joppert, talk shop about differing opinions in the community about deprecating and ultimately removing language features, have the efforts of the PHP-FIG gotten anywhere in 3 years, and Manuel's PHP Zeitgeist report.

Programming With Anthony - Iterators
Programming With Anthony - Mediators
Anthony Ferrara back with some well made video tutorials on PHP Iterators and the Mediator Pattern, with his usual hand-written examples and clear to understand style.

Better Workflow in PHP With Composer, Namespacing, and PHPUnit
A step by step video tutorial by Jeffrey Way on the current best practices with Composer, namespacing, unit tests and everything you should be doing if you're not already.

Articles

Partial mocks with Mockery
Simon Jodet manages to perfectly sum up my own feelings on mocking in PHPUnit and then goes on to (briefly) cover the new kid on the block atoum, but gets onto the main meat of his post, a good quickstart guide to partially mocking a class with my own favourite library Mockery.

An unexpected result from the PHP manual: odbc_setoption()
An extremely short and sweet post highlighting a humorous and apt quote from the PHP odbc extension manual pages. Made me .

Google's Two-Factor Auth - Online or Offline
Chris Cornutt, or PHPDeveloper.org fame, goes step by step through the process of using Google Authenticator for multi-factor auth using his GAuth library, available via Composer/Packagist, with some nice configuration examples.

Password Hashing with Zend\Crypt
Chris Cornutt again with a nice write up of using Zend\Crypt from ZF2 to create secure BCrypt hashes in your apps.

7php: Interview With Cal Evans
Khayrattee Wasseem back with another interview with a PHP community luminary, this time Cal Evans, master attendee of conferences, host of the brilliant Voices of the ElePHPant, ex-Zender, and prolific article writer to seriously name but a few. In Khayrattee's own words he's the icon of the community, so don't miss out on this great interview.

7php: Interview With Timothy Boronczyk
Khayrattee again with another interview (honestly, I don't know how he keeps up), with Timothy Boronczyk managing director of PHPMaster.com, and co-author of "Beginning PHP 6, Apache, MySQL 6 Web Development", which incidentally my wife used to get herself through a PHP course last year (when she wasn't asking me of course). Lots of good points made by Timothy, especially about PHP strengths (although I think Cal really nailed it in his interview with the community), and especially like his little emacs jab, as a vi[m] user myself.

Quick Debugging of PHP Scripts in Emacs with Geben and Xdebug
Speaking of emacs, Oracle's Christopher Jones gives us this quick step-by-step guide to getting xdebug debugging working in everyone's favourite LISP engine, ahem I mean editor.

A WebSockets, Ratchet, Silex and Redis pubsub implementation
James Morris needed to get a betting companies existing web app onto an iPhone, with realtime-or-bust speed, so polling a RESTful web service was out of the question. He ended up implementing a nice proof of concept using Ratchet and Silex to power a WebSocket connected event-driven app. If you're looking for solutions in this space and don't want to travel in Node.js territory, this might be for you.

PHP and webdriver – this time with a partial solution
Adam Goucher is a man on a mission, to get his fork of the webdriver bindings for PHP accepted as the defacto bindings by the community, and he a whole raft of reasons why we should take it seriously, along with putting his own money where his mouth is (well time, but as a consultant time=money).

Using PHP Data Objects
.net Magazine's Daniel Lewis brings us a simple guide to either using PDO for the first timer or switching your current database access to PDO. Nice and detailed with the usual mix of code and visuals.

Node.js for PHP Developers part #1-4
Interested in Node.js and asynchronous server-side JavaScript? Well Francois Zaninotto is here to the rescue, he's compiled a guide (now up to 4 parts, part 1 is here) detailing the different aspects of Node.js development and how you can relate and transition to them coming from PHP. PHPDeveloper.org has a summary and links to all 4 parts here.

PSR-Huh?
Elias Zerrouq gives us this summary of the PHP-FIG, what a PSR is, what the currently accepted PSR's are, why you should care, and various criticisms in the community, especially regards the recently accepted PSR-3.

Parsing and evaluating PHP in Haskell: Part 1
Parsing and evaluating PHP in Haskell: Part 2
Jani Hartikainen wanted to try writing a language parser in Haskell as an exercise, forgoing the obvious make-your-own-language and deciding to stick with another language he knew quite well, he decided on writing a PHP parser. This extremely well written and detailed 2 part (so far) series is not only a nice little intro to Haskell, but also for anyone interested in language parsing and why PHP is such a bloody hard language to parse (as static analyser writers have complained about for years).

How to Write an Operator Precedence Parser in PHP
A great article by Sherif Ramadan on operator precedence, the math and computer science behind figuring out precedence, and ultimately writing a parser for operator precedence in PHP, going from basic tokenisers through to lexers, and dealing with precedence overloading via parenthesis.

Functional programming in PHP
Lars Strojny gives us the skinny on function programming without breaking out the LISP or Haskell, using lambdas, closures and the great array of functional mapping builtin functions in PHP to avoid many common pitfalls of PHP dev by using a different coding paradigm.

PHP Mentoring and the Importance of the Software Apprenticeship
Elizabeth Smith brings us this weeks human interest story, so to speak, with a look at why mentoring aspiring PHP developers is so important, not just for the developer but for the mentor and the company as a whole. She's very passionate about the subject, and quite frankly I believe you should be too. Go on, read it and get involved, you know you want to.

Using Interfaces For Exceptions
Brandon Savage highlights some of the pitfalls when extending base Exception classes (such as Exception itself, or those in the SPL), how traits don't necessary help, and how he believes using interfaces is the best of both worlds.

Setting max_input_time (with data!)
Kevin Schroeder doesn't like letting so called conventional wisdom go without good reasons and more importantly real data. This time he tackles why some people suggest setting max_input_time (the time taken to parse a HTTP request body and querystring) to very large sizes. He has example code and benchmarks he ran in a small VM, along with results and some reasonable conclusions. The short of it is it depends on your use case, but if you only need to deal with small requests you can possibly protect yourself against some DoS attacks with a smaller time setting.

Zend Framework 2 Event Manager - A Gentle Introduction
Matthew Setter's gentle introduction to the EventManager in ZF2 (one of my favourite parts of the newly rebooted ZF), which drives much of the internals of ZF2, providing a event-driven interface to managing an app life cycle, as well as providing Observer and AOP style patterns. Matthew goes into a little bit of detail on why you should care and how you might use it in your own code.

Projects

atoum
Billed as a "A simple, modern and intuitive unit testing framework for PHP", this looks to provide a great alternative to the showing-it's-age PHPUnit.

Mobile-Detect
A simple library for detecting mobile devices based on HTTP request headers, such as the user-agent string.

PHP.rb
Actually a Ruby library that cross-compiles Ruby code to PHP. Why? Not a clue, but it's quite cool how it does using AST generation.

php-xxhash
A PHP extension that allows you to generate hashes using the blazingly-fast xxhash algorithm, highly useful if you're in an environment were speed matters, such as realtime streams.

csswizardry/fluid-grids
Ex-colleague of mine and CSS wizard (although I've never seen him wear his hat or get out his wand) Harry Roberts released this code for generating fluid grid designs from PHP, and has asked for feedback, pull requests and forks to make it better.

semver
Semantic Versioning is wonderful and pretty much supported by anything that needs to care about versions (including Composer). This small library allows you to parse and compare semantic versions with a nice API.

Isolator
This neat little library provides you with a proxy object that you can inject into your own objects and use to call PHP builtin functions, then during testing you can tell the proxy to mock out selective function calls rather than calling the actual builtin. Handy.

cosmos
Cosmos is a library for parsing and manipulating PHP classnames and namespace paths, which can be helpful if you want to map canonical class names (e.g. MyRouterClass) to actual importable classes or objects (e.g. \MyLibrary\Helpers\Routing\MyRouterClass).

Ubiq
Ubiq provides different methods for manipulating and parsing strings with a very readable API.

JQJobs
A lightweight asynchronous job queue manager written in PHP, similar to beanstalk, with (obviously) full PHP stack integration.

kumite
A library for managing multi-variate (such as A/B) testing from your PHP, from start (splitting users up based on criteria) to finish (gathering the results of the test).

Datum
Another DateTime extension with expressive object orientated range APIs.

Iniliq
An INI config file parser that unlike the PHP builtin parsing functions allows for multiple file inheritance, subfield values and selectors, and JSON value support.

Negotiation
Negotiation allows you to easily implement content negotiation in your app, e.g. returning different values based on supported client features and other HTTP goodness.
