Audiovisuals

Lately in PHP discuss PHP possibly getting JIT compilation and more
Regular host Manuel Lemos and guest host César Rodas discuss the latest round of discussions in the PHP core dev team about adding JIT compilation to PHP itself, inspired by Facebook's HipHop VM. Although the general consensus is that HipHop will never be merged into PHP, but some of the devs are interested in implementing LLVM based JIT. Exciting stuff. No, really. The usual roundup of PHP news was also discussed, including Manuel's own PHPClasses and JSClasses sites getting their new karma systems.

PHP Town Hall talk about upcoming features in PHP 5.5
Episode #2 of this nicely produced podcast (which somehow managed to slip under my radar until this last week) features regular host Phil Sturgeon is joined by community stalwarts Zack Kitzmiller, John Crepezzi and Anthony Ferrera, or in their own words "a Node Hipster, Beardy Python Fan, PHP Contributor and a Bristolian" discussing some of the kinky new features in PHP 5.5, and how they compare with similar features in other languages.

Articles

Debugging Connections with the MongoDB PHP driver
Derick Rethans continues his excellent series of blog posts on working with the MongoDB 1.3 PHP driver. In this article he covers the great new debugging and logging features for fixing a wonky connection to Mongo, including the vastly new and improved MongoLog class.

How Is Aura Better Than (er, Different From ;-) Than Symfony and Zend?
Paul M Jones posted this English translation of his email interview with PHP Magazin all about, you guessed it, the Aura framework's advantages over Symfony 2 and Zend Framework 2, including the decision to make Aura PHP 5.4+ only.

Aura: More Decoupled Than Symfony 2 and Zend Framework 2
In this follow up post Paul M Jones goes into the nitty gritty of using the various Aura classes outside of the framework and just how much more decoupled they are than the classes in Symfony 2 and Zend Framework 2 (both of which known for being highly decoupled from their core components, hence their use in other frameworks like Laravel).

Community News: Privates - Harmful or Useful?
The brilliant PHPDeveloper (if you don't read it, go subscribe now) covers the latest back and forth across the PHP community about use of private properties and methods in class designer, originally sparked off by this post by Brandon Savage.

9 Magic Methods in PHP
Lorna Mitchell gives a nice overview of 9 of the many magic methods in PHP 5.x from ye olde constructor to lesser used serialisation and object copying hooks.

Adding DTrace Probes to PHP Extensions
Following on from David Soria Parra's post on tracing PHP code using systemtap on Linux, this post by Oracle's Christopher Jones goes over the steps for building PHP with DTrace probe support and creating trace files for C extensions.

PHPitfalls
A nice article covering security steps and considerations when designing your PHP application for public use. A few points are a little dated (e.g. only mentioning the mysqli extnesion and not PDO) but some good points overall with solid recommendations like profiling the whatsits out of your code and this gem: 
"Rule 1 of input data hygiene: Nuke it from orbit, it’s the only way to make sure!"

Array Union Operator
Fabian Blechschmidt talks about the little-known array union operator, or as you-may-or-may-not know it the plus sign (+), and use-cases, in particular using it to fix a bug in the Magento Mage_Core_Helper_Data::removeAccents method.

Performance Calendar: Using NGINX, PHP-FPM+APC and Varnish to make WordPress Websites fly
A thorough write up by Tobias Baldauf on his Nginx+PHP-FPM+APC+Varnish stack tweaked to rafters for Wordpress, but plenty of pointers for PHP app stacks in general, especially the Nginx and PHP-AFM configs.

Build Automation with Composer Scripts
Good overview by Ignatius Teo of Composer's script hooks with an eye on automating build tasks by running scripts at different point during projection installation/update/uninstall. Some good pointers here on achieving the sort of things you might normally use Phing or Pake for, but with the awesome of Composer.

Le renouveau de PHP (English: The revival of PHP)
A French post by Clement Herreman on the various advances in both PHP itself, and supporting PHP projects like frameworks and Composer, that make developing in PHP fun again, or in his own words "rendant de nouveau le développement en PHP rapide, agréable et fiable".

Projects

Guzzle
This is an HTTP client library, reminiscent of requests in Python-land, with an elegant API that lends itself perfectly to working with RESTful webservices.

ErrorExceptions
A nice example of the power of PHPs error handlers which allows you to catch and handle system errors such as E_ERROR, E_WARNING, E_NOTICE etc. as real exceptions. By author Anthony Ferrara's own admission probably not production suitable, hey there's not even a license, but a good example of what can be done.

Leggio
A markdown based static blog generator, ala Jekyll, but with plans to be even more feature loaded and powerful.

Ray.Di
This dependency injection library is fork/extension of Aura.Di adds Guice-style (a Java dependency injection framework) DI to your PHP projects.

phpDaemon
An old and well established project that implements co-operative asynchronous long running PHP processes using non-blocking callbacks, ala Node.js, Gevent, etc. with a worker model ala Erlang and Go. Very interesting project that's worth a look if you want to implement something like a websocket server but don't want to use a mixed platform.

tinyl10n
Handy util for detecting the user's locale based on HTTP headers.

Msgfmt
A class based .po files to .mo file (used in i18n translations) converter based on Joss Crowcroft's original php.mo implementation.

goodby/csv
A memory efficient evented CSV file parser that reads a CSV file line by line and performs operations based on the line, like SAX parsing but CSVs, and with nice multibyte support so it won't choke on those UTF-x chars.

 

RFCorner

A quick glance at some of the features and changes considered for PHP via RFCs.

Autoboxing
Under discussion, this is a suggestion to add a new magic function __autobox which would be called at runtime whenever a primitive type, .e.g integer, string, boolean, etc. is accessed like an object, this function if provided by the user would return an object wrapper, written by the user, allowing you to treat the primitive as an object with methods and properties, e.g. imagine popping a value off an array by calling $array->pop(). Looks like this idea has mostly garnered nays from core devs due to the possibility for cross-project conflicts, lack of consideration for the Spl objects already provided, and possible slow downs caused by it.

Jsonable Interface
In the works, this is one of those ideas which you've probably hit on yourself if you've ever tried json_encode-ing an Object. The gist of it is to add a Jsonable interface which your class can implement and provide a __toJson magic method which would return a base type like a string or array etc. that can encoded safely by json_encode, and whenever you pass an instance of the class to json_encode, yep you guessed it, the magic method would be called and it would all just work. ++good. No patch as yet but doesn't look like there's any objections, so fingers crossed for a future release like 5.6.

Allow arbitrary expression arguments to empty() and isset()
Coming in 5.5, this simple feature allows empty() and isset() to accept non-variable expressions, e.g. the result of an inline like ($foo + $bar)/6 or a function/method call (with the return value being passed directly to empty or isset for testing).
