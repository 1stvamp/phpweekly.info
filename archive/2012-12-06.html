PHP Weekly

Issue #1 for 6th December 2012

Well, that escalated quickly

When I first announced my desire to curate a weekly PHP related newsletter I expected a couple of signups, perhaps a pity-subscribe or two from friends, but I never imagined the fantastic response I got, picking up over 200 subscriptions before the first weekend had passed.
Wow.

My sincere thanks to everyone reading this inaugural issue, I wouldn't be doing this if not for you and the great community that surrounds PHP and makes it such a comfy place to develop, learn and share.

A few people have asked me if they can help out with the weekly and if so what can they do?
What can you do?
Go and make something! Make anything, it doesn't have to be cool, but if you pour a little bit of time, effort and your amazing mind into it you'll find it probably is rather cool.
Go forth, make things, share your knowledge, help others, cast your code to the wind, and use a nice license. It'll make my job so much easier, and it'll make the community a better place to boot.

Audiovisuals

Programming with Anthony: PHP references (video)
Sent to me by a non-programmer friend this quirky video is the third in a series of programming explanations for beginners and novices. Anthony Ferrara records his hand drawn notes as he takes them, with a clear voice over explanation, in this instance going into a nifty description of references and copy-on-write pseudo-references in PHP 5.x - probably old hat to many like myself but if you haven't come across it before have a look.

Voices of the ElePHPant podcast interviews..
..with Moses Ngone
..and Mike Dawidowicz
If you've not come across Voices of the ElePHPant before stop what you're doing and get thee to a podcast-subscribing-thingumie. Cal Evans interviews, mostly every week, a different person of interest in the PHP community, from user group organisers, to prominent project developers, core developers and more. His recent interviews with user group leaders are especially good for insights into the intricacies of getting a local user group off the ground and supporting a small community.

Lately in PHP podcast: Should PHP be copying Java?
As well as the usual round up of PHP news regular host Manuel Lemos and guest Michael Kimsal discussed the often polarised discussion of PHP borrowing more from the Java world, including features and practices, spurred on by the recently rejected proposal to introduce Java-like generics to PHP.

Articles

Speeding up PHP-based development with HipHop VM
The clever chaps at Facebook's engineering dept. have been very busy with HipHop, in fact they created a whole new implementation of it (HH itself being an alternative implementation of PHP that compiles to machine-code-compilable C++) that works using Just In Time compilation. Not only is now as fast as it's C++-compiling cousin, but it's steadily getting faster. This reminds me of the Facebook backed experiment to get an implementation of PHP running on the JIT-toting PyPy (Python on Python) from earlier in the year.

Why some people hate PHP
Phil Sturgeon gives a nice roundup of answers to his Quora question - why is it (some) people hate PHP? Most of the reasons there are clear answers to, some may surprise you, and some I still feel there's more we as a community can do to answer/"fix".

First impressions of the Laravel framework
Write of using the Laravel framework for the first time by the University of Kent's Matthew Bull. Laravel is been on my to-try list for a while so it's good to get such a detailed first look.

Micro-optimizing in PHP
We've all done, put a hard reference in that big data loop, switched from a print to echo because ZOMG print is returning something, and the usual hard lesson to learn is to just don't do it because PHP is actually pretty good at runtime optimisation and doing the right thing. The folks from Developer Knowhow give a short roundup of places where micro-optimisations can make a difference, and the intersect with good coding practices.

5 Things Codeignitor Cannot Do (without a rewrite)
Phil Sturgeon again with a list of shortcomings of the Codeignitor framework, most of which already provided by other popular frameworks, and how they could be accomplished..but only with major rewrites in the framework itself.

The Lesser-Known Contributors Who Are Trying To Make PHP Good Again
A slightly older post by Daniel Nill on packages and PHP features making the PHP development ecosystem more rich and pleasant to work in.

PHP Objects Under The Hood
Sherif Ramadan goes deeper into PHPs object and class system than you're likely to have gone before, from how PHP managers symbols in global and local symbol tables to what actually happens when you instance that class into a local variable, or how calls bubble in that stack when accessing this property.

Mongo is dead, long live MongoClient
If you're working with MongoDB in PHP then heads up, the Mongo class is being deprecated and replaced by the nifty new MongoClient class, Dereck Rethans gives us the skinny on why and what you should do to move to the new API. His equally handy follow up post details connection handling with the new class.

The state of meta-programming in PHP
Good write-up by Lars Strojny on meta-programming APIs and toolkits in PHP and third party packages, from the Reflection API to token parsing, AST, runtime symbol adjustments, opcode parsing and meta-data handling. Short story: there's lots you can do with regards reflection and introspection but actual runtime meta-programming is still in the realm of possibly dangerous extensions or memory/CPU heavy parsing, but there are interesting advancements coming up, and for most people it doesn't matter with proper packaging from Composer and meta-models as part of most frameworks, like Zend.

Probing PHP with systemtap on Linux
PHP has had support for DTrace probing points since 5.3, but systemtap is a similar solution for Linux. David Soria Parra recently added support to PHP 5.5 to enable tracing on Linux and details in his blog post how to use systemtap with 5.5.

24 Pull Requests
Not PHP specific but fun nonetheless. This is a project to encourage people to contribute to the world of open source during the festive season by submitting a pull request (you know on GitHub, where everyone seems to be these days) every day up to Christmas. We may be 6 days into that schedule already but there's nothing stopping you catching up, not only is it fun, but it'll help you improve your skills, give you more things to add to your CV, and give something back to projects you probably use every day.

Projects

SimplePHPEasyPlus
A tongue in cheek Mathematical addition library billed as "A simple, pragmatic numeric operation api written in PHP. It does addition." Yes you head right, it does addition, in the most contrived but entirely object-orientated approach possible. This is a nice shot over the barrel at all the OO APIs that we see every day making the simplest of tasks an exercise in Java/C#-like pain. Let's do less of this hey and just make stuff happen? APIs and patterns are important, but not the answer to all things.

FuelPHP Event Package
Framework independent event library, much in the vein of Node.js's EventEmitter.

tagger
A simple and elegant markup generation library that makes great use of fluent interfaces.

nagdash
Nice looking Nagios dashboard billed as modern replacement for Naglite2 (doesn't rely on parsing the much maligned status.dat file that Nagios pushes metrics into) by the same author.

MockServer
Symfony2 mock HTTP server, useful for unit tests that rely on HTTP requests where you can't mock out the client itself.

 

RFCorner

A quick look at what's happenin' in PHP RFCs including implemented changes coming in 5.5.

Generators
Coming in 5.5, generators are a concept that will be familiar if you've done any Python work (or even C#). These are effectively iterable objects that act like arrays but can yield values dynamically on each iteration. Yes you could do this before with the Iterable interface and ArrayAccess, but this removes much of the boilerplate and makes it more intuitive. This API also brings the possibilities of coroutines, but that could be an entire email in itself.

Simplified password hashing
Coming in 5.5, we're finally getting a simple inbuilt framework for generating and verifying secure passwords hashes, using either Bcrypt or PBKDF2. If you want to use this in earlier versions of PHP Anthony Ferrara created this API compatible shim.

try, catch, finally!
Coming in 5.5, the finally keyword has been added to ye olde try-catch blocks to allow you to execute a block of code along with a try block even if an unhandled exceptions occurs. Good examples of this are cleaning up resources regardless of exceptions being thrown inside the catch blog.

list() in foreach()
Coming in 5.5, allow you to expands sub-array values, for example:
foreach ($users as list($firstName, $lastName)) { echo $firstName; }

Enhanced Error Handling
Under discussion, an ongoing discussion about providing a single API for defining how PHP should generate errors and then handle them, including the concept of allowing a single container for all errors to be created, and even how errors are made (e.g. getting real exceptions for inbuilt errors rather than the inbuilt types etc.). I'm all for this and hope an accepted solution is worked out. Also related: Errors as Exceptions.

Function call chaining
and
Object instantiation and property/method call chaining
Under discussion, these similar RFCs would allow us to do things like foo()(); and new foo()->bar();

C# style property accessor/mutator syntax
Under dicussion, if accepted we would be able to define dynamic accessors and mutators for properties (like we do with __get/set*) using a simplified syntax. ECMAScript 6 (next version of JavaScript, effectively) is also getting this.
