News & announcements

PHP 5.5 beta 2 released
Mostly bug fixes from beta 1, see the NEWS file for more details.

Help Ed "Funkatron" Finkler give his Open Sourcing Mental Illness talk EVERYWHERE
Ed has a wonderful talk addressing mental health issues in an honest and open way, which like many of us he has personally dealt with his whole life, and he'd like to present it to as many people as possible in as many places, so he's crowdfunding those efforts. The PHP community managed to band together and hit his target goal in the first 24 hours, but the more money we throw at him the more places like smaller events and user groups he'll be able to get to. Go go go!

Symfony 2.1.9 released
A few bugs in the 2.1 branch fixed, including some security issues, so if you rely on Symfony2 or use components which do get upgrading! You can see the changelog for more details.

MongoDB PHP Driver 1.3.6 released
This release allows commands to be run on secondaries if they can be, were as previously they would always be run on primaries.

phpMyAdmin 4.0.0-beta3 released
This looks to be a bugfix release in the 4.0 betas, no idea what's in it but if you're trying out the 4 series betas you should probably upgrade to this new one.

Elefant CMS 2 beta 4 released
The 4th beta release of this excellent CMS marks the 2.0 branch being feature complete, it's just bugfixes from here out, so if you're an Elefant user or want to give a new CMS a try, give it a looksie.

MODX Cloud to become “SiphonLabs”
One of the biggest commercial entities behind the MODX CMS is effectively spinning off into 2 groups, the current owner behind MODX, MODX LLC, and a new company called SiphonLabs who will continue to support the development of MODX as well as customers but also drive development of a cloud platform supporting use of MODX and it's components (and adoption of said platform). The other big news from this is that they'll be pushing on with Wordpress and possible Magento cloud hosting, which  brings a wider range of software to the table for existing customers as well as putting another big name Wordpress/[Magento?] SaaS/PaaS provider out there.

 

Audiovisuals

DC4C: Worst Case Scenario slides
Dr Keith Casey Jr's talk from Day Camp 4 Developers #5 all about avoiding problems with your presentation, mitigating issues that arise, or making the best of it when things go awry, from practicing and recording your presentation, to being prepared for technical difficulties, to the sort of things you just can't avoid. I'm still going through all the DC4C videos myself so can't comment on Keith's actual talk, but there's plenty of tips in the slides to help you out.

The PHPPodcast episode #1 – Cal Evans
Unfortunately can't comment on this maiden episode in the new (hopefully) regular PHPPodcast series, by Joe Stagner, but the first episode is an interview with Cal Evans who never fails to deliver when it comes to discussions about the PHP community and PHP itself, so recommended just for that.
Update: I've been listening along while writing this issue and the first episode is great! There's lots of backstory on changes in Microsoft's stance on PHP back in the day from Joe, Cal's views on the community of the years, and some interesting thoughts on the new relationship between mobile frontend development and backend development.

FLOSS Weekly episode #244 – CakePHP
Regular co-host Aaron Newcomb (covering for regular host Randal Schwartz) is joined by co-host Randi Harper. They interviewed CakePHP core dev Mark Story, discussing how CakePHP positions itself against other web frameworks, focussing as it does on convention over configuration, ease of installation and fast initial time to first app. Some good discussion about removing magic in CakePHP 2 for improved performance, security improvements and what Mark thinks sets Cake apart.

Becoming A Better Developer - Programming With Anthony
A short entry in this great series from Anthony Ferrara with a slightly different, more free form, format. He talks a little about possibly the greatest skill as an IT professional you can ever pick up and expand upon: how to figure out and problem and find resources. As Anthony puts it nobody will ever expect you to know everything, but you better know how to find it.

/dev/hell Podcast Episode 30: It's Episode 30, You Guys
Chris "GrmpyProgrammer" Hartjes and Ed "Funkatron" Finkler return with another eclectic Developer Hell episode, covering topics from JavaScript, to testing (of course), lots of JavaScript testing, speaking and Ed's aforementioned Open Sourcing Mental Health crowdfunding efforts.

Articles

If PHP Were British
A tongue-firmly-in-cheek what-if post by Dave Child supposes a new version of PHP taking on British mannerisms, politeness and old British Empire ideas of class. Funny stuff, and in case you fancy jumping on the HN bandwagon of calling this out as an American misunderstanding Anglo's, Dave is a fellow Brit.

7php Interview With Lineke Kerckhoffs-Willems Co-Founder Of ProTalk.me
Another excellent interview from our friend Khayrattee Wasseem with talented and omg-I-can't-believe-she-has-time-to-do-get-all-that-done Lineke Kerckhoffs-Willems, discussing her involvement with PHPwomen.org and ProTalk.me, as well as general topics such as tips for improving your PHP skills, time management and QA.

Where have all the core devs gone?
Lukas Kahwe Smith points out that the majority of core developers who got Symfony2 out the door are now primarily leading/involved with other projects, or have mostly fallen quiet, leaving lead maintainer Fabien Potencier to cover and prioritise (in secret, but by chance no purpose) all pull requests to Symfony, and decide what goes into new releases. With this break down of organisation around the project it means there's a lack of openness and clarity, and Lukas thinks a good solution to this would be to bring in the role of release manager, just like PHP core (and many other open source projects). Sounds like a good idea to me, Fabien isn't entirely convinced by all of Lukas' points but he fully on board with improving the situation, so watch this space.

Etsy: There and Back Again: Migrating Geolocation Data to GeoNames
John Marc blogged about how Etsy have improved their geolocation and location aware features by moving from a closed provider to the GeoNames open dataset, implementing internal services in PHP to consume/process this data and provide locality info based on various points. They have the code, for mapping this data to MySQL tables and performing geo queries on them, up on GitHub.

Abstraction In The Real World
A typically good post by Brandon Savage on not only the value of abstraction in your code, but lots of concrete examples of different types of abstraction showing how they actually help maintainable, flexibility and often comprehension.

Lukas Smith: For the very first time: Hello UK!
Another write up of the previously mentioned PHPNE, this time by Lukas Kahwe Smith, showing not only more evidence that it was a fantastic event, but also highlighting his favourite parts from talks, workshops, and talks down at the pub, with a predictably higher slant towards Symfony2 topics.

First Phing Plugin
Lorna Jane Mitchell goes over developing her first Phing plugin, from initial creation to usage. This is a topic I have quite a bit of experience with and Lorna nails this first steps guide very well, although I must admit one thing I'd like to see in a future post is how to effectively test Phing plugins, maybe I should just write it myself finally.

Pick PHP Requirements for Packages Responsibly
Phil Sturgeon gets right to the point with this post, arguing that it is incredibly irresponsible to release open PHP libraries, with a view for others to use them, that require new "shiny" features in 5.4 (and even 5.3) that actually add nothing to the library itself, the main example being using [] instead of arrays just because they look more awesomer, because then your library requires an entire upgrade of PHP to be used in another project, and by extension people need to be careful about the dependencies they choose for their projects for this same reason. He goes on to argue against the "but it boosts adoption of new PHP versions" fallacy, and to tell people to just write for an older version. I would add that this of course really only applies to open projects, if you're writing your companies business app in 5.4 because you have it on your servers and your team like the syntactical sugar, then bloody well have at it.

PHP 5.5 Preview: New Password Hashing API
Andy Podner gives us this well written intro to the new password hashing API coming to 5.5, how to use it, what it brings to the table, and the main benefits for using it.

Predicting Random Numbers In PHP – It’s Easier Than You Think!
An excellent write up by Pádraic Brady on why the use of mt_rand is not only never sufficient with cryptographic functions but indeed should be avoided for all but the most trivial operations, because it can actually be predicted due to the lack of entropy in the seeds used to generate random numbers, which entirely voids their usefulness.

A First Look at React
React is one of those extremely interesting new projects in PHP that opens up so many possibilities you probably didn't know could ever be possible. This article by Alexander Cogneau tells you all about the Node-esque async framework that React provides, how to get it, setting up an initial server/application, and where to go from there. As Igor Wiedler, creator of React, says himself it's probably not quite ready for production but if you're interested in one of the possible futures of non-blocking async in PHP then this might be for you, and Alexander does a good job summing it up.

The same app 4 times: PHP vs Python vs Ruby vs Clojure
Adam Bard wrote this post comparing a simple example application, which replaces non-nouns from a form post with the words "nurble", written in 4 different languages and platforms. I was of two minds whether to include this post as the PHP example is definitely not how the majority of PHP devs would write it and Adam even admits to having not even tested if it works because he didn't want to install Apache+PHP at the time. I decided to include it mostly for the other language examples (and links at the bottom of the article to examples in other languages/frameworks by third parties).

PHP is not dead
David Corbacho wrote this follow up to a 2007 article, PHP is dead... long live PHP! by Dries Buytaert which showed concerns about PHP 5 adoption at the time. In his article David shows that since then PHP 5 adoption as surpassed PHP 4 by orders of magnitude, backing up the claim with statistics and graphs from various dependable sources, interestingly including StackOverflow questions as a metric which I'd not thought of before (showing the rate of developer adoption by how many questions are being asked about that stack).

NetTuts+ Round Table #1: Should Exceptions Ever be Used for Flow Control?
This is a first in a series of roundtable discussions by NetTuts+, edited by Jeffrey Way, with regular contributors Csaba Patkós, Pavan Podila, Gabriel Manricks, Jonathan Cutrell, Aaron Kuzemchak, Ben Corlett all discussing the use, and abuse, of exceptions in flow control. Some great back and forth here in a subject that has many pros and cons, which rarely fails to polarise developer opinion. Personally I fall on the "it makes sense in some situations but it depends on your abstractions" camp, read along and see where you fall (and a try-catch pun in there somewhere but I'm tired so I'll skip it this time).

On empty return values
Larry Garfield expands on his brief tweet "Protip: If your function returns an array, and its no-data return is FALSE rather than an empty array, YOU'RE DOING IT WRONG!", by explaining what he means by "returns an array", in most cases this refers to the fact your function should return a collection (e.g. an iterable) as it's valid value, so empty values should be empty collections, and error cases should be exceptions. +1 on much of this, although I'd like to point out that the description of a function that returns a single item seems to get lost about half way through. Personally I'd say if your function returns a single item (even if that item is an array or iterable object) then a null or false value is perfectly acceptable, an an array() would indicate an actual value returned.

Projects

array_column
As I mentioned last week array_column is coming to PHP 5.5 but if you'd like to use it  in <5.5 projects then array_column author Ben Ramsey also published this user-space implementation, and even if the inbuilt array_column is available (e.g. you run this under 5.5) everything will continue to work as usual.

Wisdom
A domain name availability checking API, which uses the Whois component from React under the hood, allows you to see if a domain name has been registered or not, and has a very nice workflow for adding new TLDs.

JSONCharInputReader
An API for processing JSON data as a stream with event triggers, which means it's much more efficient for handling very large JSON datasets, and you can even get away with only processing the part of a stream you actually need (e.g. you've got 20mb of JSON but you only care about a couple of fields in the first 1mb, imagine doing this over 1000s of datasets and you begin to see the advantages).

AzaPhpGen
An extremely interesting library that could be described as a serialiser, but is actually a code generator. It takes PHP variables, objects, classes, functions (as closures) etc. and attempts to dump the underlying PHP code to a string. It is already very feature complete and has lots of potential use in caching, as well as template based code generation.

Laravel 4 Map
This handy utility for Laravel 4 reveals the underlying internal static interfaces, and those in dependencies such as Symfony2, that the higher level API wraps in facades (object instances, using DI).
