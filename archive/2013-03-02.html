News & announcements

PHP 5.5.0 Alpha5 released
Missed this last week, latest 5.5 alpha, mostly as bugfix with minor features added, expect more fixes and less features as we approach release (maybe). See the NEWS file for details.

PHP 5.4.12 and PHP 5.3.22 released
10 bug fixes for the 5.4 and 5.3 branches, not greatly exciting but that's usually a good reason to upgrade early. See the ChangeLog for details.

Symfony 2.1.8 released
Lots of fixes, including some for nasty security issues, if Symfony2 is part of your stack and so often is even if you're not using it directly then look at updating now, the CHANGELOG has specifics.

Symfony 2.2.0 RC3 released
Probably the last release candidate before 2.2 goes gold next week, lots of fixes and tweaks, again if Symfony is part of your stack then you may want to start testing 2.2 now.

Zend Framework 1.12.2 released
ZF is still one of the most used frameworks out there, especially in enterprise environments, and this release has a massive 50 bugfixes and enhancements, especially if you use Zend_Service_Twitter to intereact with the Twitter API, which has been updated to support the recent Twitter API changes so it will continue to work after March.

Articles

On predictable PHP release cycles
Lukas Kahwe Smith wrote this well reasoned argument against delaying the 5.5 release for 2 months to include the newly open sourced Zend Optimizer+ (which passed voting for inclusion), as he points out while PHP hasn't had a very predictable release cycle in the past, having one is an advantage to the community and this delay just makes things worse. While I agree for the most part I've seen a lot cross-community and cross-library issues with stringent release cycles like the one Canonical has with Ubuntu, and it can be hard to strike a balance between predictable and shackled.

On PHP-FIG
The prolific Matthew Weier O'Phinney shared his thoughts and feelings on the PHP-FIG's effectiveness and weaknesses, and his choice to step down from the organisation he founded. While he sets out a substantial list of issues with current FIG practices, and hurdles yet to overcome, his main reason appears to be that he's tired of dealing with it all and feels he has things he'd rather be getting on with in his life. Power to you Matthew.

Rock On, Refactor, or Re-roll?
Andy Podner wrote up his recent experience at work, tackling an issue many of us developers face: live with it, refactor or restart from scratch, and the struggle of choosing the pragmatic approach or the "fun" path. He finishes up with a very balanced solution, something relevant for everyone.

C-based Web Frameworks for PHP
Fortrabbit co-founder Ulrich Kautz goes into the pros and cons of C based PHP frameworks, how they provide massive amounts of speed and memory advantages, but can be a pain to debug if you're not familiar with C. Good write up and some excellent points, and one that I feel definitely backs up the notion that these framework driven functions are in such high demand by real developers that perhaps more should be provided by the engine itself.

Next big thing in PHP
Florin Razvan Patan not only gets his crystal ball out and tries to predict the next big feature set or shift in attitudes, but he also covers similar gripes a lot of people are expressing at the moment, most famously Phil Sturgeon, in that real developers are asking for features to be included in core that would make their lives easier and many feel those concerns are being ignored because of a vocal few. As well as this Florin reckons PHP is on a slow death march, with less contribution, performance problems, etc., and that there should be a big community push to make PHP6 a break from the past, with a cleaned up and more performant engine at it's core.

7php - A Comprehensive Interview With Zurmo CRM Tool
Another excellent interview from Khayrattee Wasseem, this time with Zurmo CRM co-founder Ray Stoeckicht. They talk about Ray's background, why he and Jason Green created Zurmo, what makes it different from other CRMs, and how they have impressively avoided the traditional "open core" approach that others like SugarCRM have embraced. Good stuff.

Functional Programming and PHP
A solid overview of functional programming basics by Shameer C, with lots of PHP examples, covering how you can achieve the fundamentals of functional programming such as immutability, recursion, referential integrity and pure vs impure functions, as well as discussing the pros and cons and why you might like to consider embracing functional programming for your next project. +1.

Use the Accept Header to Set Your Return Data With Zend Framework 2
Jason Fox details a nice little method in ZF2 for changing how return data is formatted or encoded (e.g. markup, XML, JSON etc.) based on the Accept HTTP header sent by the client, which works especially well with frontend JS apps, as well in public APIs and means you don't have to provide every URI mapping under the sun for each return format.

Taiming repository classes in Doctrine with the QueryBuilder
Anne from EasyBib describes an elegant solution for extending a Doctrine repository class with a generic get*/findAll//findBy*/filterBy*​ API that is maintainable and extensible without there being a mess of edge-case methods cluttering up the class. Very clever and well written, would be good to seea generalised class open sourced for this. 

Why objects (usually) use less memory than arrays in PHP
The brilliant Nikita Popov wrote this quick but detailed gist on the internals of object vs array memory usage in PHP core, how objects aren't always heavier than arrays due to how the hash table for an object is constructed for non-dynamic properties in >=5.4, how this means objects especially with large data structures will often be smaller overall (despite a large base footprint) than arrays and faster to access. There are some great takeaways from this at the end, read on for more.
 
The Definitive PHP Style Guide
A tongue-oh-so-firmly-embedded-in-cheek alternative to the PSRs, with such wonderful nuggets as:
  • All classes, functions and methods MUST have a line of ///// after the opening braces to delimit the beginning of them. You're probably best off using the full 857 character limit just incase someone views it on a hi-res screen.
  • Comments MUST be right-aligned, using tabs, to improve readability.
  • All variable, constant, class, method and function names should [be] EXACTLY 8 characters long.
Perhaps not one to try at home folks.

Projects

AzaThread
A nicely done library for running code in "threads", well not actually threads, it emulates them by forking extra processes and using inter process messages to control, inject and return data. Very nice API, with an inline fallback for systems that don't support forking.

Go
Not to be confused with the Google programming language or the aspect orientated library, this is a simple build tool inspired functionally by Ruby's Rake.

Stash
A cache abstraction library, at it's core provides a dependency-injection-container-like API but with support for various caching backends to allow persisting a cached value across requests, currently supports: files, APC, Memcached, and sqlite, but it is easily extendible with new drivers and also allows using multiple staggered drivers.

MtHaml
HAML for PHP, inspired unsurprisingly by the Ruby library, with first-class Twig integration,

Purl
No, not the language with a camel on all the O'Reilly, this elegant library provides methods for parsing a URI into it's constituent parts and manipulating them.

PHPSandBox
A framework for running PHP code safely in a sandbox that works by parsing, validating and if need be cleaning the code, even swapping inbuilt/global functions in place with wrappers. Nicely done and if you're in the position of needing to execute code from a source that isn't your own this could be for you.
