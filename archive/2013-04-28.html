News & announcements

PHP 5.5 beta 4 released
A few bug fixes, which is both promising and expected so late in the beta release cycle, it looks like we're on track for the release candidate on May 9th. Now is the time to get testing if you've not begun already. See the news file for more info.

Elefant 1.3.5 released
Another bugfix and feature packed beta release from this extremely promising CMS. A key new features is the addition of role based access control which is a definite must for any serious CMS, especially for enterprise use, and the Elefant devs seem to have gone about it in the right way. See the changelog for more info.

Audiovisuals

PHP Podcast episode 2 ♥
Zend's Joe Stagner is back with another episode in this interesting new podcast series. This time he is joined by another Zender, Adam Culp, to talk about the PHP community, how Zend is trying to engage more with the overall community, differences between freelancing and working for a company like Zend, and more.

/dev/hell episode 31
The infamous Chris Hartjes (♥) and Ed Finkler (♥) return to shoot the breeze about anything and everything they can, covering the usual topics of PHP, JavaScript, testing, conferences, and frameworks. Listen out for talk about frameworkless frameworks (as Rasmus would put it) and lightweight frameworks.

Cloud Out Loud - Future of PHP: Drupal
David Shafik talks to well known core Drupal-ers Angela Byron, Dries Buytaert and Larry Garfield to talk about one of the most popular CMS-come-frameworks on the web today, what's changed in Drupal 8 and what is to come.

Build RESTful ZF2 Applications ♥
The slides from Matthew Weier O’Phinney's webinar (free to watch, but requires registration) on building RESTful web apps with Zend Framework 2, Matthew starts with a good overview of the principles of REST itself before launching right into practical examples.

Articles

Simple logging of ZF2 exceptions ♥
Rob Allen wrote this nice little piece on using Zend\Log along with ZF2's event listening to create and attach an application logger that catches your exceptions for you. Very neat and possibly useful even if you're not running a pure ZF2 app.

Performance benchmark of popular PHP frameworks ♥
A great speed comparison (requests/sec), by Lukasz Kujawa, of thirteen of the most popular PHP frameworks. As Lukasz says it is incredibly difficult comparing apples for apples when it comes to frameworks but he's tried to level the playing field as much as possible with a basic project that uses the standard configurations for each. The great take away from this isn't the unsurprising news that Slim is fast, being a microframework, but the finishing paragraph about the real purpose of frameworks, to boost development. Speed can always be boosted through other methods.

Testing Laravel Controllers ♥
A great and very thorough step by step guide by Jeffrey Way on testing Laravel controllers in isolation. As well as testing Jeffrey covers refactoring your application core using inversion of control, making testing extremely clean and easier to do.

Sandboxing Untrusted Code With PHPSandbox ♥
Elijah Horton goes through using his own PHPSandbox library to test "untrusted PHP code" in a sandbox from the executing system PHP, which is especially good if for some reason you need to execute user provided code (something you should usually try to avoid at all costs but there are some use cases, like with sites like ScraperWiki), I've covered PHSandbox box briefly before but I'll say again that it's a very cool project, and things get really interesting with the ability to override entire functions in the executed code, including builtins, with your own.

PHP is meant to die
An interesting piece on the suitability of PHP for long running processes. Before we get into the fact that actually yes PHP can be perfectly used for long running processes, especially since a lot of GC cleanup was added in 5.x, I'll say now that I agree with the fact that PHP's basic model is that it is optimised for a single short lived request. Rasmus has stated before that he sees that as PHP's strength in being horizontally scalable with a shared nothing architecture, and afaik every feature, change and optimisation to PHP is developed with that in mind. Another point that is dead on is that we shouldn't really do background tasks inside requests, even though there are tricks to do this and a few frameworks implement in their core, it's a bad idea and should be avoided (disclaimer: I recently did exactly this too.)

The Epic Tale of Moving to HipHop
tuenti's Daniel Paneda (♥) made this excellent write up on their move to Facebook's alternative PHP implementation HipHop, which you may remember statically compiles your PHP to C code (which is then compiled to a native binary), and can deliver significant speed advantages over cached PHP opcode interpretation. Lots of details here on what they had to change and why, mostly dropping newer PHP features and APIs due to not being implemented in HipHop yet, and also the parts of HipHop they improved themselves to get going. I'd love to hear a similar story for HipHop VM, anyone out there done it in a production env?

20 Point List For Preventing Cross-Site Scripting In PHP ♥
Pádraic Brady wrote this handy twenty point todo list that covers what you either need to do or at least consider in order to protect yourself against XSS attacks. Some good points here, and while yes a lot have pointed out that at the end of the day "don't trust anyone, armour everything" is a succinct way of putting it these finer points will help most people get there if they don't know where to start.

Bonnes pratiques Symfony2 : notre condensé! ♥
This langue française piece by Thomas Jarrand is a great list of take-away best practices from Symfony Live 2013 for developing with Symfony2. Some of these are excellent points for any developer to take on board, not just Symfony ones

Write secure templates with Latte ♥
A great write up by Jiří Pudil on how to use the Latte templating library, that comes as part of the Nette framework, to write highly secured templates for applications. They're secured via context aware escaping, with macros and helpers that make not just writing code easier but also allow you to improve escaping by providing increased context. I'd not come across Latte before, and while it's very similar to Twig in many regards, it does have a few nice unique features, especially in the automatic nature of it's escaping.


Projects

morafabio/vagrant-php-development-box ♥
Another base Vagrant project loaded with a base viewee configuration to create a reusable box, as well as Chef cookbooks to install the usual LAMP tools you could want in a dev VM.

ZF2 Restful Module Skeleton ♥
A very nicely written and full featured base ZF2 project to get you going writing a RESTful app with the ZF2 RestfulController package.

Toro ♥
A richly featured yet extremely slim routing framework for PHP, which has one of the nicest APIs I've seen in one of the multitude of loosely coupled route libraries now available.

Doctrine Annotations ♥
Doctrine famously uses annotations embedded in the docblock for methods and properties to perform extra runtime metadata handling, such as database operations in different contexts. This is the library that performs the docblock parsing behind annotations so if you fancy doing something similar yourself, here you go.

SetnGeti ♥
Speaking of docblock annotations this library does something similar by automatically adding accessor and mutator methods for properties if you add @get/@get to the property's docblock.
