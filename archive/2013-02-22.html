Audiovisuals

Design Patterns - Programming With Anthony
Anthony Ferrara back with another excellently executed technique-revealing video, this time joined by Larry Garfield of Drupal fame, all about design patterns and how embracing their use, or at the very least being aware of them, can increase your code maintainability and reuse. Everything is backed up by well thought out concrete examples, and cooking tips.

Beautiful Models in PHP
The slides from Brandon Savage's PHP UK 2012 talk on modelling data in PHP, well worth a glance over, if only to understand the difference between modelling data in your application and storage (e.g. database backends).

PHP Town Hall Episode 4: PHP's Vision, Beards, and Cake
Regular podcast hosts Phil Sturgeon and Ben Edmunds are joined this time by CakePHP core dev Jose Diaz-Gonzalez and bearded-Laravel-whisperer Zack Kitzmiller, to talk the talk mostly about the future direction of PHP, reactions to/opinions about Phil's slighty-incendiary blog post, and comparisons with other languages like Python and Ruby.

Articles

PHP OOP Test
Matthew McConnell started this interesting discussion on reddit about testing a candidates programming skills before an interview, and even shares the test he's trying. This is one of those necessary evils when interviewing that is difficult to find the right balance with, and there's plenty of good advice here.

A Journey on Avoiding Nulls in PHP
Michael Nitschinger is on a mission to never type if ($foo === null) { throw blah; } again, and he's reached a clever solution by borrowing the Option concept from Scala and Google Guava Java framework. A good write up of uses for the option/optional pattern and a simple port of the Google classes included.

Don't use PHP libraries with known security issues
Fabien Potencier announces SensioLabs new service, a PHP Security Advisories Checker for Composer projects. This is a brilliant idea, with website, RESTful web service, and command line tool, that works by checking the versions of dependencies in your composer.json against SenioLabs CVE repository, and allows you with both manual and automated ways of checking for insecure deps in your projects. Bonus tip in the comments from Jordi Boggiano of Composer fame on using hooks to automatically check for issues on composer.lock update.

Preventing CSRF Attacks
A typically great write up by Anthony Ferrara on CSRF attacks and their prevention in PHP apps, with each stage of prevention broken down with examples, and most importantly why you should do it this way.

Would this be a dumb idea for PHP core?
An interesting proposal (not a formal RFC, he's just putting it out there so to speak) by Kevin Schroeder to add some method to PHP to allow keeping a global application state cache between PHP invocations (between requests/PHP interpreter executions) in order to minimise application bootstrapping times, e.g. autoloading classes, core object instances, globals etc. Using something like APC, or the newly open sourced (and possibly coming to core) Zend Optimizer+, opcodes are cached between interpreter instances, however a lot CPU cycles/memory is burnt just loading initial app state and this could definitely be a big performance boost, but there are valid concerns.

7php Interview With PPI – The PHP Meta-Framework
Khayrattee Wasseem interviews Paul Dragoonis again, this time about the PPI Framework, from the why he started to the project, to what exactly a meta framework is anyway, and why you might just want to make PPI2 your next framework of choice. I love meta frameworks and PPI2 is pretty much the only major player as far as PHP meta frameworks go, and definitely one to watch/use.

Setting Up A Staging Environment
A detailed write up by Gabriel Manricks on creating a localised staging environment, using one of my favourite tools Vagrant, that emulates your PaaS of choice (in the case of the article Fortrabbit) and allows you to test your project in as close as possible an environment to your production deploy target. Some nice tips and tricks, especially creating git hooks in a local bare repo to mimic git push based deploy processes.

Wrapping a PHP class for easy method chaining
Stefano Azzolini gives us a simple method and wrapper class for enabling method chaining on existing instances, which if you're not familiar with the concept you've probably used it with jQuery chained methods.

Dependency Injection Container Question
Matt Frost has thrown out this tricky question about using DICs with dependencies that require extra dependencies via parameter injection, and whether there is place for both paradigms at the same time. Using both together doesn't sit well with Matt, and I can see why, personally I'd tackle this using containment wrappers to allow use of shared dependencies within the DIC that can be passed on instantiation to other dependencies from within the wrapper. The main problem then becomes not just reinventing a registry pattern. My 2 pence so to speak.

Testing Like a Boss in Laravel: Models
A detailed and thorough write up by Zizaco Zizuini on testing Laravel 4 models using some of the better third party packages to make your life that bit easier, one favourite tip of mine being the use of an in-memory database (using SQLite) to speed up test execution.

Closures, Lambdas, and Anonymous Functions
An excellent article by Andy Podner on lambdas, the distinction between lambdas and closures, and where you might use either in your code. Lambdas and closures are awesome and I honestly believe PHP devs need to use these common CS constructs more, especially given the new ability in 5.4 to use $this in closures, there are many more possibilities for making our instances more dynamic.

Looking back: Sunshine PHP 2013
Jacob Mather wrote up his experiences at last weeks Sunshine PHP, with some understandable name dropping, and tips on how to get the most out of a good PHP conference.

Projects

I Feel PreTTY
A lovely little library for formatting your output from command line apps to come out all pretty with colours and layouts in the terminal.

httpful
Yet another HTTP library but httpful has a brilliant chainable API that definitely makes web API calls mucher nicer. Here's an idea for free, libraries like Httpful and Guzzle should share a common HTTP lib, similar to how whole frameworks are beginning to share common decoupled components.

composerdoc
A helper package that adds a utility to your Composer package to auto-generate dependency documentation.

archer
A framework for standardising unit tests and continuous integration runs against a set of predefined conventions, that if you follow give you some great benefits, like using the best quality tools and automatic features.

EmailReplyParser
A PHP port of GitHub's excellent Ruby email parser. If you work with email responses from external sources such as users you should seriously consider this utility, it's based on one of the best parsers of it's kind and this seems to be a well done port.

RFCorner

A quick look at interesting RFCs in discussion or possibly even coming to a future PHP release.

Trailing comma function args
This extremely simple proposal by Sara Goleman (yes, that awesome Sara Goleman) to allow function calls to have a trailing comma similar to arrays, e.g. str_replace('test', 'blah', $foo,);
An excellent idea that adds more consistency to the language and has several of the same advantages as trailing commas in array definitions. Yes it adds backwards incompatibility but this is really just a problem for open source projects aiming for older versions in that they'd have to lint for trailing commas, but they have to do that anyway now!

array_column
This proposal by yet another well known PHPer, Ben Ramsey, is at the voting stage, and based on the votes (which seems to have ended last month?) will definitely now make it into a future release. This adds a new array_column function, that acts very much like the PDO fetchColumn method, and allows you to pluck a specific column of data from a multi-dimensional array, which is a fantastic idea that I'm sure will reduce a lot of peoples code.
