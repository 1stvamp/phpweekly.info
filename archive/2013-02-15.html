The giveaway that wouldn't die, part II - the revenge

Last week I attempted to give away 2 prizes (see last weeks intro for the skinny) to mark reaching over 500 subscribers, well that didn't go so well because I made a classic newbie mistake when it comes to giveaways: I forgot to check if the recipients actually wanted them.

So let's try this again, if you'd like one of the two prizes I mentioned, email me telling me which one you'd like or that you'd like both.

News & announcements

Zend released Zend Optimizer+ as open source
In order to further the ambitions of the much talked about proposal to integrate the Zend Optimizer+ opcode cache into PHP core itself, Zend have released it fully as open source on GitHub, under the PHP license natch. It's definitely looking good that this will make it in, although I've seen a lot of comments on reddit hoping for it to become a PECL package either way.

php|tek conference schedule posted
Just as in previous years php|tek is shaping up to be an awesome conference, and they've posted the schedule featuring some extremely well known and interesting speakers on the roster.

Lone Star PHP call for papers
Lone Star PHP is another great conference that will be happening in June in Dallas, and they've put out a CFP, if you think you might have some interesting and relevant to talk about, why not head over and submit a proposal?

Propel 1.6.8 released with tonnes of bugfixes

 

Audiovisuals

Arduino "Basic Blink" Using PHP
Admittedly not an awful lot happens in this video, and yet it's rather impressive at the same time. Thomas Weinert shows off the first demo of his Carica IO project which implements the Firmata microcontroller protocol in a non-blocking PHP library. What does it do? Right now it makes an LED on an Arduino blink, but the possibilities are great.

Austin PHP - REST Best Practices
A well presented talk by Twilio's Dr Keith Casey Jr covering the key elements of a RESTful web service, gotchas and best practices.

Articles

Composer Stability Flags
Igor Wiedler, of Composer, Symfony2 and Silex fame, put together a very succinct description of how minimum-stability and version/stability resolution in Composer works, as well as whitelisting on a per package basis different minimum stability using flags. This is one of those things, as Igor states, that comes up on IRC all the time (in fact I've answered it in #composer-dev a few times myself), and one to read up on if you're confused by why certain packages complain about different versions like this.

Debugging Silex applications just got fun
Fabien Potencier wrote a little about how both Symfony2 and Silex gain a lot from using the Symfony HttpKernel component, despite one being a full stack framework and the other being a microframework, but that you sometimes miss certain developer utilities and functions that ship with a full stack like Symfony2, but one such component the Symfony web profiler can be added via a Silex provider.

Logging with PSR-3 to Improve Reusability
A nice primer and tutorial by Patrick Mulvey on using the PSR-3 logger interfaces to improve component reusability in your code, as well as using the adaptor to allow incompatible loggers to still work with the PSR-3 interface, reducing the coupling of your code with a specific component, and the amount it needs to know about how that component works.

RESTful APIs with ZF2 part 1
RESTful APIs with ZF2 part 2
Matthew Weier O'Phinney is writing this series of how-to's based on a workshop he and Rob Allen gave at PHP Benelux 2013, all about the ins and outs of creating a RESTful API, with an introduction to the Richardson Maturity Model, hypermedia and the Hypertext Application Language, followed by real world cases and how to achieve them using Zend Framework 2. Good stuff.

The Mysterious PHP RFC Process and How You Can Change the Web
An excellent and fairly exhaustive list of dos and don'ts for creating your first RFC/proposal for extending PHP, with simple advice like mailing internals if you're new, including an implementation with your proposal for maximum effect and my particular favourite advice strive to do better than current RFCs.

Parsing XML With SimpleXML
SimpleXML is one of the truly wonderful builtin libraries in PHP which I miss whenever working with another platform. Sandeep Panda gives us this small walkthrough on how to use it to parse a simple DOM and get data out using the object property notation.

PHP5.5: Try/Catch/Finally
Joshua Thijssen goes into detail about what the new finally keyword does in PHP 5.5 and why you might need it, as well as what it's advantages are over how you may have performed cleanup from an exception before 5.5 (clue: finally will always execute before the try-catch block is left, meaning it will even run before an exception which isn't caught by the block). Good to see a greater level or detail and real examples being shown.

My Love / Hate Relationship With PHP Traits
A good write up of what traits in 5.4 really bring to the table, how they can be used in a sensible fashion to extend an object functionality without much fuss, and a few warnings about how they can be abused (although personally I'm not so fearful of them, but then I spend most of my time in completely dynamic runtimes were multiple inheritance and "sideways" object extension are normal).

Generating secure cross site request forgery tokens (csrf)
Kevin Schroeder put together this post on generated CSRF anti-forgery tokens and how many people are doing it wrong by rand() and uniqid() builtin functions as they don't use enough entropy when generating random output and so often produce predictable results, which you definitely don't want in an anti-forgery token. Kevin describes instead using the better openssl_random_pseudo_bytes() to generate tokens, and that you don't really need a hash of your token either.

Of Creating User Groups and Physics
Some typical well written thoughts by Rafael Dohms on creating and continuing to grow a user group, with an excellent analogy to thermodynamics.

Free Control Panel for Zend Optimizer Plus
This author decided to hack together a quick and dirty control panel script for the newly open sourced Zend Optimizer+, and also shared a couple of insights to the API and what it does/doesn't tell you about the internal cache.

Microframework using anonymous functions
Emanuele Minotto created this one-script microframework using mostly just anonymous functions as registered callbacks. Putting this here rather than in projects as the gist reads more like a how I did it guide with some nice details and tricks.

External processes and PHP
Giorgio Sironi goes over the various ways of invoking an external process from PHP and you can gain greater controls over processes and output using the Streams API.

Projects

GitList
An online Git repository viewer with a lovely design and some nice features.

PHPHint
Both a web service and open source project that provides a web based PHP linter to check your code quality and catch any nasty unnoticed issues.

stamp
Based on the Ruby library of the same name stamp allows you to parse and display datetimes without needing to remember a single arcane parameter or date() call. Very nice.

Gaufrette
Gaufrette is a filesystem abstraction library which means in simple terms that you can work with files and file structures using an abstracted API in your app and not actually care where those files are stored or how, so when you need to change to a different file store (e.g. switch from local to Amazon S3, or just to a different location locally etc.) you can quite easily. You can also add caching on time of slow stores, amongst other great features.

Faker
Anonymising live data for testing? Creating fake user data? Stress testing etc. etc. Well Faker might just be the lib for you, does what it says on the tin, generates sets of Fake data from given constraints, with some very nice preloaded types of fake data set.

mongounit
An extension for PHPUnit that adds some handy utilities for testing MongoDB stored data.
