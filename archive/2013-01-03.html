PHP Weekly

Issue #5 for Thursday the 3rd of January 2013

It's 2013, where's my damn jetpack?

Another rotation round the sun and another PHP Weekly for inbox.
If you thought the festive week itself was quiet, then New Years was practically silent, but fear not intrepid reader there a few morsels to wet your appetite for the year ahead.

As everyone seems to be blogging about their New Years Resolutions, but no one ever follows through with them, here are some New Year Suggestions:
  • Release more code.
  • Submit some talk proposals.
  • Be excellent [to each other].

News & announcements

Pop PHP Framework 1.1.1 released
Some CSRF/Captcha form changes and a request parsing method added, see the changelog for more details..

Articles

12 TDDs of Christmas
John Cleary set himself the challenge of doing a test-driven-development kata a day (from the 26th of December), and encouraged other to join along on Twitter with the hashtag #12TDDs, it looks like it was a smashing success and some interesting solutions produced by people too.

Become a ZCE in 2013
Lorna Mitchell is encouraging you to make 2013 the year to become a Zend Certified Engineer, and she throws up plenty of resources to help get you there (some free, some cheap). I've been a ZCE since 2007 and I'd strongly suggest looking into it.

PHP equal operator ==
Gynvael Coldwind gives us this very thorough and in-depth write up of what actually happens when you compare the different internal types and user defined objects with the double equals (==) operator, tip: it may not always do what you think it will.

7php: Interview With Musketeers.me On Taking Over PHP|Architect – Goal Is To Make Existing Processes More Efficient With Fresh Bloods
Khayrattee Wasseem is back with another great interview with PHP community luminaries, this time with the new owners of php|a the team from Musketeers, on their motivation for taking over from Mark Tabini and co, the challenges of taking on an already established property with such a solid community standing, and what's in store for the magazine in the future.

Why Lithium is Worth a Look
Andy Podner was a man with a plan - rewriting an older project as a standalone app to expand into something else in 2013 - but what framework to use? Deciding to use the new year to explore something new he gave Lithium a try, making sure it has support for all the development tools and practices he relies on in CodeIgniter. A good write up with both pros and cons of using Lithium.

SOLID - The D is for Dependency Inversion Principle
Freek Lijten with a nice overview of Dependency Inversion Priciple, a development practice which avoids replication and reduction of untestable/unmockable code by defining known interfaces for objects and using dependency injection for each usable component, which means you can write, use and test the API for application and it's components, not the algorithms and business logic within.

Getting off the island in 2013
Larry Garfield put out this call to the Drupal community to get off the island in 2013, and by that he means the inclusive Drupal-only-or-GTFO island that is slowly emptying by way of using non-Drupal modules (via Composer, 'natch) in Drupal itself, and steadily releasing decoupled Drupal code to the wider community, but he wants to see more. Drupal devs attending non-Drupal conferences, interacting with the wider PHP community and wider web dev community, giving back and bringing in more wisdom to Drupal. Many other projects and communities with inclusive mindsets could do to take note of this excellent advice.

Symfony Components: Sometimes Decoupled, Sometimes Not
Paul M Jones runs down the good guys, bad guys and anti-heroes of decoupled components in Symfony2. It still stands that some components are completely decoupled and can be used at will in your apps, but 4 core components aren't decoupled at all and have mandatory requirements, while Paul reckons 5 components have a debatable decoupled status at the very least.

Creating a PHP OAuth Server
A solid tutorial by Jamie Munro on creating an OAuth 1.0a provider using oauth-php, from using Composer to install it manually from Google Code (in lieu of an actual package on Packagist) to setting up the necessary MySQL schema, HTML forms and PHP controllers.

First Impressions: Cloud9 IDE + PHP
James Fuller wrote up his first impression of using the PHP integration in the excellent Cloud9 IDE (which you may recognise in a small part from using the integrated ACE editor on GitHub), with a similar conclusion to my own, including the shortcomings (especially as a fellow vim user).

Personal Thoughts On The PSR-3 Log Proposal
Stuart Herbert weighs in on the polarising PSR-3 debate with his personal thoughts on the matter, in short he isn't against the LoggerInterface standard, he just thinks the standard could be a lot better and in his post details how he'd like to see the proposal improved.

PHP CLI usage survery
A very short survey by Tom Hudson on developer PHP CLI usage (e.g. development of command line apps with PHP), he already had a pretty big response, but take a minute to fill it out so he can get an even bigger picture.

Projects

serverdensity/mendedstring
Some blatant self-promotion with this one as I just released this code on New Years Day based on migration code I wrote at SD a year ago. The MendedString class allows you to detect and decode single byte encoded multi byte strings, e.g. UTF-8 characters encoded (even multiple times) with utf8_encode. I'll be releasing a blog post soon on why we needed this and how we used it.

Patchwork
A library that uses stream handlers in PHP >=5.3 to allow you to override user functions, ala the runkit extension using pure PHP code. Use with caution, but it just might be that magic bullet you need to test an external library you don't have control over and can't using dependency injection with.

jyggen/curl
A lightweight cURL wrapper with multiple request support. Could be handy if you don't need the full power of Guzzle.\

lootils/geo
Library for dealing with geodetic calculations, for example finding the distance between 2 points on the earth.

snowscript
Another language that cross-compiles to PHP, inspired by Python, Lua, Coffescript, Go and Scala.

PHP-Parser
A PHP (<=5.4) parser written in PHP, with the aim of making static code analysis/manipulation easier to implement and execute.

pChart
Initialised chart generation library with an OO API.

phargs
A very nice little, and elegant, toolkit for creating command line apps. Nice to have something similar to Clint in PHP-land.
Combine it with docopt.php for even more awesome.
