News & announcements

PHP 5.5.0 beta 1 available
It's finally here, opcode cache and all, first 5.5 beta for your QA purposes. This one is important to get testing folks as it's now a clear road to release. See the NEWS file for a list of additions, or read through the first 3 articles in the articles section below for various write ups.

PHP 5.4.13 and PHP 5.3.23 released
A few important bug fixes, include security issues, so get those package managers warmed up. See the ChangeLog for more details.

Laravel 4 beta 3 released
Laravel 4 is shaping up to be an awesome release, and this beta adds a raft of new features and fixes, with excellent code coverage in the tests to boot.

LoneStarPHP 2013 schedule released

Symfony Live Paris 2013 programme released

 

Audiovisuals

Laracon: Eloquence Emerging (video)
A break down by Taylor Otwell on the primary goals and idioms behind Laravel which is rather eloquent itself.

Laracon: Better Testing In Laravel 4 (video)
An excellent guide by Jeffrey Way on improving test usage and coverage in Laravel 4 applications, with great general advice for testing thrown in for good measure.

Laracon: Per Your Request (video)
Kenny Meyers takes us through what happens during a full request cycle, and how the request/response is handled by Laravel, from library calls to patterns and execution.

DC4D: Baby Steps to Ballroom Dancing: Getting on the Path to Speaking Success
Day Camp 4 Developers #5 looked amazing this year, with some amazing talks, not least of which was this guide by Elizabeth Naramore on overcoming "glossophobia" by making sure you don't throw yourself into the deep end, starting with topics you know well and audiences you can relate to easily. Good stuff.
 
DC4D: How to propose a talk and get it accepted
Laura Thompson gives us the low down on creating a great talk proposal, getting your initial ideas across to organisers and avoiding common mistakes.

DC4D: Preparing to Speak
A thorough walk through by Lorna Mitchell from 3 months prior to giving your talk, turning your abstract into that perfect presentation, making sure all your support material is covered, and knowing when you change things.
 

Articles

Introducing Array_column() in PHP 5.5
Ben Ramsey shows off his addition to PHP 5.5, the array_colum() function, which others may know as a pluck function, allowing you to easily select subvalues from an array structure, regardless of dimensions, with control over returned indices. An excellent addition to the language, and good introduction from it's author.

List of new features in PHP 5.5
A comprehensive list of new feature additions to PHP 5.5 by Nikita Popov, author of several of them, complete with links to RFCs and the author's name.

Using PHP 5.5's New "OPcache" Opcode Cache
A write up by Christopher Jones on building 5.5 with the new opcode cache and setting it up, along with the caveat that the new OPcache extension doesn't include a user data cache like APC does, and links to projects that add one as a separate extension.

Symfony Docs Hack Day Needs You on March 30th
Want to contribute to Symfony2 in one of the best ways possible? Several of the core team members will be holding a hack day on IRC (#symfony-docs on Freenode to be precise) on March 30th, 9am CET to 5pm EST, to strive to improve the documentation. Just hop on IRC, discuss things with people on there and get submitting pull requests for issues on GitHub. It's a great way to give back to the community and get your name added to improvements in a major open source project.

PHP version 5.3 is now the most used version, just ahead of 5.2
W3Techs, who collection tech stack statistics from various companies and individuals via surveys, reckon despite the fact 5.3 is at end-of-life this month, it has only just overtaken 5.2 adoption by about 3%. To be honest, I can believe it, we all know adoption is slow on any language/platform, especially one so closely tied to host providers who often just stick to current major releases, and continue to get patches from vendors such as RedHat for years after end of life. A commentor also points to Wordpress's stats which still show 5.2 ahead at over 60% for Wordpress installs.

PHPNE 2013 Conference Report
PHPNE 2013, the first large PHP conference held in the North East of England, looked like an amazing day and I'm sad I missed it, but I've been hearing great things. Adam Smith has written this excellent report on all the talks and other events during the day, which makes me look forward to attending next years (here's hoping).

26 Ways to Show that PHP Can Be Better Than PHP
Manuel Lemos of PHPClasses fame goes through a very extensive list of "missing" PHP features that can be provided by, or closely emulated, using userspace PHP code, and for each gives an example of an existing implementation. Some very clever libraries and workarounds listed here, although I must say one of my biggest bugbears is that most of these aren't in any form of public source control (e.g. GitHub, BitBucket etc.), making open source collaboration much more difficult.

Look mama, no databases
Michelangelo van Dam wrote this nice article on decoupling your code and your unit tests in turn from your database operations, by extracting out those operations to replaceable methods which can be overridden when mocking, meaning you can run your unit tests independent from a database instance or requiring any connections to one. A solid write up on a good principle for application design not just testing.

7php Interview With Stefan Priebsch A Co-Founder Of The PHP Consulting Company aka thePHP.cc
The prolific Khayrattee Wasseem brings us another 7php interview with a PHP community luminary, this time Stefan Priebsch, author, speaker and co-founder of thePHP.cc. The usual array of personal tips and recommendations from books to software and conference lay herein, personally I usually like to compare notes with other developers especially the parts that show they're just human (I have no time left these days ever Stefan!)

Using Stackato for PHP Applications in a Private PaaS
Andy Podner takes us through setting up and initial use of ActiveState's Stackato, which bills itself as a "free micro cloud", in other words an integrated suite of management applications and platform software (e.g. a mini lamp stack with apps for automatically managing and deploying applications via the stack, like a mini Heroku, AppFrog etc.) An intriguing idea in itself, Andy goes step by step through initial setup and an initial app, and covers ideas on how you might use it in your own setup.

First Serious Attempts with PHPUnit, Composer, and the Omniture API
Justin Carmony takes us through lessons he learnt using test driven development to to write a wrapper for Omniture's RESTful API, which after a weekend of hacking he has it almost complete and up on Packagist. Some good lessons for everyone here.

Some thoughts on using array_* with closures
Ashish Dattacovers one of my favourite subjects, using anonymous functions with the extensive set of array_* array manipulation functions built into PHP, which often produces clearer, cleaner and more precise code, and also allows more slightly more dynamic control over manipulation. One thing to note however is that if you're calling manipulation functions many times and over lots of arrays you'll usually get more performance out of a statically defined callback as opposed to a runtime defined closure.

Orno series part 1: Building a RESTful API
Phil Bennett takes us through developing a RESTful webservice with the Orno\Di dependency injection library and loosely coupled MVC component, which despite having covered before on PHP Weekly I was not aware just quite how awesome it is, allowing you do buildup application patterns using data structures (the good old trusty array) and leverage dependency injection to componentise each part of the app. Really excellent stuff, and one I'll be playing with in the future.

The dire state of WordPress
James Shakespeare probably isn't going to cause many gasps of surprise with this post, but he does succinctly describe a lot of problems with using, deploying, developing for and developing on Wordpress as it stands today. I've developed some heavy content managed sites on top of WP, a platform definitely not designed or suited for it but we still manage to wedge into and on top of, and James makes some good points about how WP is still one of the best options, even if it does have lots of disadvantages, and let's not ignore the install base alone means we need to seriously think about improvements, not just migration. I will add one point, if you're following this weekly and your primary exposure to PHP is via Wordpress, you're going to want to go to look at other resources right now. Please. :)

PHP courses on Codeacademy need you
Codeacademy recently added much awaited PHP courses to their list of guides, but the PHP ones are still incomplete and they'd like the community to help fill them in.

Projects

Bernard
A very nicely implemented message queue written in PHP and driven by Predis/Redis for push based message transports. Also includes based task running functionality backed by message queues.

Rhubarb
An incomplete but extremely well designed PHP client library for the Celery distributed task framework, which itself is primarily Python based. Although the non-AMQP backend support is lacking (interfaces are there but not yet implemented by the looks), this project already provides a very feature rich API.

PHP-CSS-Parser
A fairly complete CSS3 parser, with manipulation functionality.

 

PHP HTML5Lib
A full PHP implementation of a HTML5 tokenizer matching the WHAT-WG specifications. Put this together with the CSS parser above and you have pretty extensive web scraping/filtering toolset.

pulsar
A fork of the Symfony class [auto]loader but with PHP 5.1.3+ compatibility. Has several advantages of many class loaders, for example APC or XCache class map caching if available.

Phar Update
A lovely API for creating self-updating Phar archives, e.g. an archive which automatically updates itself from a URL whenever a certain new version spec if available.

APCu
A project to separate out APC's user cache into an extension on it's own, as the new opcache in 5.5 doesn't include a user cache.

Yac - Yet Another Cache
Another shared user cache project to run alongside other opcode caches, but this one is lockless by design.
