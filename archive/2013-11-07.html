News & announcements

mt_rand() cracker released into the wild
Long story short: don't use mt_rand() for seeding anything, ever, the OpenWall project's new php_mt_seed is a very powerful implementation of their proposed mt_rand() seed cracker, and from various reports it works a treat. If you're a budding pen tester take a look, if you're a dev, use something better. (Caveat: technically if you're using Suhosin mt_rand is more secure as it is re-seeded on every call from /dev/urandom but you should not rely on this.)

Expectations RFC enters voting
The Expectations RFC defines a new backwards-breaking API for the oft-forgotten assert call which will make unit test a little bit nicer to write, and certain blocks such as switches and functions with unattainable sections easier to define with a standard exception. Voting has commenced and it's looking good for this making the cut.

GoPHP5: The Revenge
GoPHP5 was the record breaking push in 2008 to get the top PHP apps and libraries to drop PHP4 and finally move to 5, followed by many of the big PHP hosting providers. It worked and was instrumental and the PHP core team end-of-life-ing 4, thus pushing the majority of big companies, agencies, platforms, and pretty much everyone else into the world of 5.x rainbows and unicorns. Larry Garfield (♥), one of the leading voices behind the original movement, has raised the idea of another push onto 5.5+, saying goodbye at last to 5.2, 5.3 and the barely wet-nursed 5.4. The discussion is being held on the PHP-FIG mailing list presumably with the idea of getting many of the big names in frameworks to lead the way. Interesting to see if this gets the full head of steam the original did.

Articles

Debugging PHP applications with HHVM ♥ ♥
Qandidate's Alexander and Emil go step by step through using one of HHVM's less known gems, an inbuilt interactive debugger, which is a must have tool for any PHP dev (if you're not using one, get yourself an xdebug/DBGp compatible debugger now). With a simple CLI switch the HHVM runner becomes an interactive debugger with all the bells and whistles, reminiscent of Python's inbuilt pdb.

HHVM with Symfony 2 looks amazing ♥
Yes indeed, it seems to be HHVM week. Hot on the heels of last week's benchmarks, Christian Stocker shares his stats from a recent non-trivial Symfony2 application developed by Liip for interacting with the Amazon ElasticSearch API, performing some business logic, and returning results to the client. The results are extremely promising for HHVM, in comparison with vanilla PHP 5.3 and PHP 5.5+Opcache, performing at up to 50% faster (RPM, response time etc.) across the board against 5.5 (which in turn outperforms it's ageing ancestor 5.3 at 60-70%).

DTracing a PHPUnit Test: Looking at Functional Programming ♥
Christopher Jones regales us with more DTrace'in examples, this time tracing a PHPUnit test suite (from the example code in Patkos Csaba's excellent Functional Programming in PHP tutorial), and printing out the call tree using a D script and the DTrace hooks in PHP 5.5. This is a very powerful method for dynamically tracing a program with runtime hooks and manipulation that you only get from something like DTrace with the powerful D language as an orchestration utility.

Introduction to Gearman – Multitasking in PHP ♥
Alireza Rahmani Khalili gives us this well written introduction to getting started using Gearman, the fast distributed task runner, from your PHP applications using the Gearman PHP extension, and even touches on using the worker API to write Gearman enabled jobs. If you think your project could do from running jobs out of band from the web service component (and nearly any project of a certain size does, eventually), then this is an great option, and Alireza's intro is a solid starting point.

Beyond Inheritance ♥
Another brilliantly well written and clear article from Anthony Ferrara, this time on the topic of the use-cases for using traditional inheritance, versus composition with traits and abstract classes (to provide implementation and contract in loosely-coupled components). He describes it far better than I could, so go read, even if you think you know everything there is to know about OOP I always find I come away from Anthony's articles with something new to chew over.

Carica Chip 101 - Controlling An LED With PHP ♥
I previously featured Thomas Weinert's astounding project to do Arduino hardware control from PHP, by first creating a non-blocking IO library, an implementation of the Firmata protocol used to communicate with the microcontroller in the Arduino, and finally this culmination of his efforts, a PHP powered web server that controls an LED on the 'druino. *claps* Well done, sir.

A Peek At Aura v2 -- Aura.Dispatcher ♥
Paul M. Jones details the process, both thought and practical, involved in extracting the Aura v2 Aura.Dispatcher package from three different v1 packages that handled dispatch of some variety, from URL routing, to CLI param parsing/passing, and internal method invocation. This is a perfect example of deduplification, decoupling and refactoring, with a healthy dose of future proofing and API scaling when he discusses the API for how the Dispatcher can be used from both a micro-framework or fullstack-framework.

PHP 2.0: A Review in Retrospect ♥
Melissa Beth wrote this lovely, and in many places tongue-in-cheak, retrospective of the nearly forgotten PHP/FI 2.0's manual. That pretty much sets you up for the whole piece, if you can't stand even the most well natured of prods at your language of choice you'd better look away, because let's face it if you ever used 2.x or 3.x like I had the chance to, you know there were some massive warts. As Rasmus most famously stated "It was never intended to go beyond my own private use."

Open Source Mapping with PHP and MongoDB ♥
A neat little tutorial by Ryan Jarvinen on using Red Hat's OpenShift PaaS with PHP and MongoDB to do some cool geospatial hacking, in the example he grabs some open US National Parks data, parses it, pushes it into MongoDB complete with geo data, and quickly gets a simple web app up and running to display the data on a nice map. You can also see a live running demo, and fork the code on GitHub.

Symfony2 - Service Data Caching Using Annotations ♥
A cool little Symfony2 bundle and write up by Steven Lewis over at Epiphany, which simplifies the caching of data from API calls, using a clever combination of Symfony2's service tagging system, compiler passes, proxy classes and the Reflection API to override service calls at runtime based on phpDoc annotations and provide returned data from a cache if present or from the upstream API if not. Nicely done.

Array manipulation in PHP, the correct way ♥
An easy to digest post from Mike Curry to round off things. This one is most likely teaching many of you to suck eggs, but if you weren't aware of some of the more functional ways to do array manipulation in PHP then Mike covers them here.

Laravel Weekly ♥ ♥
Last but not least I wanted to give some space over to fellow weekly I discovered recently, from the team behind the Laravel community portal, Laravel Weekly is a great little weekly mailer with links and info for one of my favourite frameworks. Their GitHub repo is worth a look as they keep all their past issues not only in the final/sent HTML, but also in the original markdown, nice.


Projects

Hawk ♥
A nicely made implementation of the modern HTTP authentication scheme, hawk.

composer-script-utils ♥
This package provides, as yet, 3 very handy utility methods for working with paths inside a Composer script, and providing interactive updating of PHP INI config params.

iniscan ♥
One of my favourite utilities of late, by the infamous phpdeveloper.org man himself Chris Cornutt. iniscan scans (as well as showing with formatting) a PHP INI config file with a set of rules, and highlights any possible security flaws to deprecated options or bad practices. Invaluable.

samsui ♥
A handy (in some situations) utility with a very clever name. It provides a factory API for generating a tonne of objects with preconfigured attributes/fields, e.g. for auto-generating test data.

Flysystem ♥
A filesystem abstraction library that allows you to use a common API for interacting with "files and directories/paths" while swapping and changing underlying vendor drivers, from the filesystem, to FTP, S3 and Dropbox. It's under active development, has a very nice API, and something I think many projects could benefit from using.

PHP-VCR ♥
Based on Ruby's vcr gem, and similar to packages like nock in Node.js, this wonderful library allows you stop writing out mocked external services. Just stop. Instead you make real API calls, record them with VCR, and have the calls "played back" (I hope don't stretch the tape, zing) in your tests. Magic.
