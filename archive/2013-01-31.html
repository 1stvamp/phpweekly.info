Articles

Comfortable PHP Editing With VIM
Tobias Schlitt has a very thorough PHP dev environment setup in my own favourite editor-come-IDE vim, which he has shared on GitHub, and also blogged about various snippets he uses. Great stuff.

1.5x … 9x faster queries with PHP and MySQL 5.6, really?
Ulf Wendel dives into the soon to be released MySQL 5.6's Memcache protocol support which effectively runs a version of memcached with MySQL and allows you to perform basic key-value queries with the binary TCP/IP or socket memcache protocol. It turns out doing this is quite fast, with as much as up to 9x speedups on certain queries. As Ulf says this works well in projects where you're basically doing simple key-value queries, and adding a caching layer like this can even speed some more complex queries, but use with caution. Remember there is no silver bullet for databases.

Intro to PHP Comments and Docblocks
Andrew Podner covers the often grey world of commenting your code, with some black and white details on the phpDocumentor sorta-standard for docblocks, how you can use it document classes/methods/functions etc., and some simple advice on general commenting.

PHP 6: Pissing in the Wind
You gotta love Phil Sturgeon's penchant for not mincing his words, this time on the subject on progressing PHP with new language features and cleaning up old language warts. From recent bitchfests in the community, stubborn, blind and quite rude attitudes and responses to perfectly valid requests from real developers are sadly becoming the norm. Phil puts forward some great suggestions for how we can improve this and move forward with the language as a community, but as the comments suggest it's not going to be easy. (Possibly some NSFW language.)

7php Interview With Paul Dragoonis CTO At BestBuys.com
Khayrattee Wasseem is back with another awesome interview with a PHP community luminary, this time friend-of-PHPWeekly and fellow Brit Paul Dragoonis, who shares the highs and lows of organising a tech conference, how he runs BestBuys.com, his love for the PHP community, tips about being a better developer, and much more. As usual check it out.

Composer: Speeding up class autoloading
Patrick Allaert proposes a very elegant solution to speeding up class autoloading in Composer beyond the option --optimize-autoloader by creating PSR-0 (for those packages which use PSR-0 for a namespace mapped file structure) mapping symlinks to class files so that the autoloader just has to point directly at were it thinks the corresponding file should be on the filesystem. He asks the Composer project leaders if this could be baked in as an extra option, but checkout the comments for more thoughts and possible issues.

GeoLocation with FourSquare and Google Maps tutorial
Paul Dragoonis over on the PPI Blog has this well put together tutorial on geolocation using FourSquare, Google Maps and of course the PPI framework, with an example project repo on GitHub, and lots of snippets for generating maps and doing other geo ops.

Dependency Injection with Pimple
A clear concise tutorial by Rakhitha Nimesh on using the Pimple dependency injection container to start using the DI pattern in your apps, encapsulating logical components in a reusable way and getting rid of those nasty global configuration painpoints.

Setting up a PHP development environment on a VPS
Aaron Pollock takes us through the process of creating your very own dev environment on a VPS for various well explained reasons, e.g. always having access to your environment, not caring about opening ports on your machine, having a single IP to test public APIs (especially useful for webhooks and OAuth return addresses) etc. Although some of these can be quite easily mitigated with a local VM solution like Vagrant, Aaron makes some good points and if a VPS env is the way for you he provides a solid guide.

When To Write Bad Code
With all the recent code hate on the web Brandon Savage reminds us that not even can start their project from a point of perfection, be they inexperienced, in a rush or just don't need to, not all code is or should be perfect, but he goes on to explain how that "bad code" that actually got written can be refactored, and when either point makes sense.

Projects

dependency.me
A website and open source project that analyses the dependencies in your Composer package and tells you if any of them stale and out date, including provides a cool little TravisCI-like status badge that you can include in you README.

scalar_objects
This is an extension by the prolific Nikita Popov that allows you to register handlers for scalar types (strings, numbers, arrays etc.) so that you can then perform operations on them using method calls, e.g. $string->length() rather than strlen($string). Very very cool.

EasyRdf
A library with a simple OO API for generating and manipulating RDF documents.

 

handy
A data model API for migrating data between different models, such MySQL tables.

deployer
A standalone or Laravel4 deployment app with simple configuration, worth a look if you don't fancy setting up something more complicated like Puppet, Chef, Salt etc.
