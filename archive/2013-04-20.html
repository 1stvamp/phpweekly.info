Audiovisuals

PHP Town Hall episode 6
Another great episode from regular hosts Phil Sturgeon and Ben Edmunds, joined by PHP celebs Paul M. Jones, Anthony Ferrera, and regular guest and fellow snake charmer Zack Kitzmiller. With 50% of the room being PHP-FIG members they mostly talk about Paul's recent PSR-X proposal to improve the PSR-0 package loading standard, and also about the wider significance of the PHP-FIG's work for the whole of the PHP community.
As usual contains swears, actually lots of swears, so avoid if you don't like that sort of thing.

Lately in PHP podcast episode 34
I'm afraid I haven't had a chance to listen to the latest episode so I'm effectively restating what it says on the site: regular hosts Manuel Lemos and Cesar Rodas talked about a better PHP voting process, new releases, updating PHP with minimal downtime and "how brilliant are some developers for creating pure PHP components that overcome PHP limitations without upgrading to a newer PHP version."

Articles

Traits are Static Access
A very thorough tear down by Benjamin Eberlei on why using traits to extend your classes can be just as hazard for code maintainability and testing as using tightly coupled static methods everywhere. He argues that traits create similar code smells to lots of static method use and have issues such as not being mockable and providing global state. An interesting point of view on traits that I hadn't considered before and I'd be interested to hear more peoples opinions. Avoid them completely? Or find better ways to use?

Salt, Hash and Store Passwords Securely with Phpass
A simple walkthrough by Mike Dalisay on how to use phpass to correctly hash and storage passwords in your app. Mike covers the basics here with some simple code examples, so I'll just point out that phpass also provides many other utilities for edge cases in password use so you'll probably want to look at the docs, and Ryan Chouinard has a nice Composer package you can use.

About Symfony: Stability over Features
Core Symfony maintainer Fabien Potencier has written up what he thinks should be the main focus for Symfony2 development over the coming months, yes you guessed it, stability. Stability means a few different things to different people however, so Fabien includes a concise list of what that means to him and which parts of Symfony need some stability lovin'.

Use Dice for Simplified PHP Dependency Injection
Tom Butler wrote this nice introduction to the Dice DI library, covering how Dice's rule based API is different to how other contains work mostly with factory classes. Tom goes on to show some simple applications for this in your code, with the goal being the ability to completely swap and change any internal dependency without altering APIs or large refactors.

Define: Functional, Unit and Integration Tests
Anna Filina describes the different types of tests we all hear about every day, from what makes them different to why you would use one over the other in different situations, and most importantly why you probably need all three to correctly cover testing a full stack. 

Diving into Behat
A good write up by Giorgio Sironi on the ins and outs of using Behat to unit test your application using behavioural driven development as a guiding principal. He provides a particularly helpful list of situations when Behat probably isn't the right choice before going on to explain writing behaviour specifications for an application, and using those specs to write actual tests, as well as caveats for ensuring tests don't become hard to maintain.

Enqueue Symfony’s process components with PHP and ZeroMQ
A great piece by Gonzalo Ayuso on combining the ZeroMQ socket library with the React non-blocking event library for PHP, to create an asynchronous queue of Symfony process comonents. A clever little example with a video showing it in action. As pointed in the article and comments React isn't necessary in his example as it is actually blocking with the use of blocking ZMQ messages, but it is possible to make this non-blocking with some work and using an event loop makes the concepts slightly easier to grasp.

Generating One-Time Use URLs
Angela Hinkler gives us this tutorial on generating one time use URLs, like those often used for resetting your password, or for sending confidential information to a contact without risk of someone else being able to open it later. A nice example of doing the grunt work for handling a URL like this, but as pointed out in the comments if you do this yourself please use a library like RandomLib to generate the tokens as those used in the examples are susceptible to timing attacks.

Do we need a framework for that? Or hurry up PHP-FIG
Phil Bennett thinks the real secret sauce provided by the PHP-FIG is in providing common interfaces for components between frameworks, to allow more decoupling and more test driven practices, just like PSR-3 does for logging, and wants to see more of this. Difficult to argue with that to be honest..but people will anyway. :-)

Deploying PHP applications with Phing
A very thorough and well put together rundown by Lukasz Kujawa on using Phing to build and deploy an entire application, with a real world example that goes from performing various file based tasks to more in-depth examples such as initialising a database and generating deployment configuration. I especially like the explanation of using and overriding properties to provide context specific deployment data, something people often miss with Phing that allows you to even do context switching based on environment.

Sitepoint: What Happened when we Talked PHP with the Experts
Sarah Hawk  sat down with tutor, speaker and author Lorna Mitchell to talk about her OO course and various topics spanning PHP development and the PHP community.

7php Interview With Manuel Lemos Founder Of PHPClasses.org
Another excellent interview from Khayrattee Wasseem with a PHP community A-Lister, this time Manuel Lemos of PHPClasses.org and Lately in PHP fame. They discuss Manuel's work with PHPClasses, his early years with PHP way back, daily development practices, and most interestingly for me the Brazilian PHP community and culture which has fostered great cooperation and spirit over the years.

Composer primer
This excellent primer by Dayle Rees covers the basics for not just building out your first package, but also some of the more trickier concepts of dependency management, autoloading and general command usage.

Automated backups to Google Drive with PHP API
Lukasz Kujawa goes step by step through using Google Drive as a backup location for general backups (such as file, data etc.), much like you would with Amazon S3 or Dropbox, from setting up an API account and using his cp2google library to power the backups.

Extending Twig Templates: Inheritance, Filters, and Functions
Rakhitha Nimesh wrote this concise tutorial on simply building filters and helper functions for your Twig templates by extending core filters and callables.

Is PSR-0 Shortsighted, or are you?
Phil Sturgeon wrote this reply to Tom Butler's PHP: PSR-0: Pretty Shortsighted, Really, which posited that there were various issues with PSR-0 autoloading. He goes blow by blow through each point with clear responses, as both as a developer and PHP-FIG member. The main sticking points seem to come down to confusion between what the PHP-FIG are trying to do (no they aren't "forcing standards" on anyone) and that PSR-0 != Composer. They seem like simple concepts by the crop up time and time again. Most people also don't seem to realise they can override autoloading in Composer if they want to define their own package structure.

Tom's PHP Commandments - Take heed
Tom Buck, living up to his site name of "Biased PHP" gives us this extremely biased list of nine DOs and DO NOTs, from things like avoiding variable variables for simplicity and clarity, to never using weak hashing for passwords or injecting variables directly into SQL strings. Some of these points are so opinionated I personally wouldn't always follow them to letter (e.g. never using CakePHP for a project, or Apache, both of which have had improvements in recent years that don't necessarily seem reflected in these "commandments"), but there are some great take-aways nonetheless.

Base Converting and Binary Operators
A great little primer by Timothy Boronczyk  on number bases and converting between bases using binary operators in PHP, not much more to say other than this is one of those things you may have forgotten from math classes or first year compsci, but you may require one day if working with numeric data, especially large data set exports from different systems.

PHPUnit vs. Phake cheatsheet
A great comparison of two of the major PHP mocking libraries, with direct examples for both for creating stubs, mocks and spies.

Reflection in PHP
A tutorial by Patkos Csaba on using reflection, or rather in this context meta-programming (which uses reflection) to inspect the properties of objects to build up APIs that don't rely on specific classes, object instances or even specific APIs to a degree, and only using some of the simpler parts of the Reflection API.

Your PHP site may not be secure
Thomas Lackemann reminds us of an old but always relevant "information leak" in the case of leaving expose_php turned on in your config which enables a special querystring to be called to display PHP info such as the version of PHP, credits and enabled modules, as well as exposing the PHP version as a HTTP header, which can both be disabled by setting "expose_php = off" in your PHP config.
Why is this a security issue? Allowing such info leakage allows a potential attacker to more quickly asses attacks that can be used against your server. Of course there always other examples of such options in your server of choice (e.g. Apache or Nginx) that can be disabled.

Interest for a PHP IDE Plugin
Subscriber and Sublime user Devon Weller is probing PHP developers and Sublime users to see if there's any interest in creating a full IDE-like plugin for Sublime that would provide full code introspection and completion for PHP such as the kind you you get in PhpStorm.

PHP Namespaces: Avoid use
Jason McCreary cautions against the use of the use keyword in PHP and advises using full namespace paths over aliases imports.
I'm not a particularly big fan of this myself, and I think if you find yourself in the position that aliasing an import into the local symbolspace can cause issues with no knowing what you're instancing at any one time and where that class originates from, I think you need to break up your namespace further, as such problems are a classic symptom of overloading your source files with class definitions.
I do heartily agree with the advice of being aware what exactly your imports are doing however and if you do have lots of conflicting class names across multiple namespaces you should stick to importing higher level namespaces not the classes themselves, as just avoiding use completely, especially large codebases is really throwing the baby out with the bath water.


Projects

Medoo
An extremely lightweight database ORM, with lots of backend support and rich featureset.

jamiefdhurst/vagrant-dev
A simple Vagrant environment that uses the shell based provisioning to setup a common PHP specific box.

Gitter
A simple and clean OO wrapper for common operations with git.

print_o
A very handy object graph visualizer for PHP for graphing dependencies between class instances. See some of the examples for a good idea what you can use this for.

openlss/lib-stream-wrapper
A lightweight wrapper around PHP's stream wrappers which provides an Interface for more easily building stream wrappers.

whoops
Taglined as "php errors for cool kids", whoops is a very nice debugging and exception handling library, which is similar to SaaS apps like New Relic or Sentry.

patch-installer
Rely on a dependency handled by Composer and need to patch it but don't want to maintain a full fork? patch-installer is an add-on for Composer that allows you to patch your Composer installed packages, while maintaining the data for the patch from your package JSON, with some limitations.

PHP types autoboxing
An implementation of SPL Types that allows you to wrap PHP scalar types with initial method calls and maintain your wrapped types (and their methods, metadata etc.) even after setting the variable to a different value.

ChromePhp
A PHP implementation of the Chrome debugging protocol used by the Chrome Logger extension which allows you to view server side debugging and logging information inside the Google Chrome developer tools right alongside your client side logs.

Nest
A very nice decorator util that allows wrap and call properties, methods and other "attributes" of nested objects using keys with a familiar array key/value notation. It's great to see more and more decorators like this crop up in PHP, like they have similarly in Python, Ruby and JavaScript.

Phinx
Phinx is an extremely powerful yet simple to use database migration framework, allowing you to migrate both schemas and data between different DB schemas, using either PHP or SQL itself.
