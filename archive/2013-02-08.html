News & announcements

Zend Framework 2.1.1 released
40 patches with a whole tonne of fixes for 2.1 in this release, if you're using ZF 2.1 you won't want to miss this release.

Idiorm and Paris 1.3.0 released - the minimalist ORM and fluent query builder for PHP
Idiorm is a tiny yet flexible ORM and and Paris is an active record implementation built on top of Idiorm. For some time they didn't receive much love as their original author had moved on, but new maintainer Simon Holywell has pushed 1.3.0 out the doors, and things are looking up, with new features, bug fixes, and a solid plan for the future.

Web & PHP magazine issue 11 is out
Latest Web & PHP magazine, released as usual for free, has some good cloud based articles, and a preview of PHP UK, which is just 2 weeks away now.

 

Audiovisuals

PHP Quick Tip: When to use the alternative syntax
Ben Everard walks us through what context he chooses to use the "alternative syntax" for IFs in PHP, e.g. mostly when mixing PHP with markup in templates, as he feels it provides a syntax much closer to the element level markup in HTML.

FLOSS Weekly #239: Concrete CMS
A favourite podcast of mine, this episode has regular host Randall Schwartz and semi-regular co-host Simon Phipps interviewing Franz Maruna, one of the lead developers behind Concrete 5, which until now hadn't crossed my radar but it certainly seems like a good contender in the highly competitive CMS space.

Lately in PHP podcast episode 32
Regular hosts Manuel Lemos and Ernani Joppert discussed MySQL 5.6 performance improvements, MySQL being replaced by MariaDB in some popular distros, the news about Zend Optimizer+ possibly being merged into PHP, and more.

Articles

Netcraft: PHP just grows & grows
Andy Ide, from the internet host metrics company Netcraft, released their stats on PHP use in 2013, and then did some neat graphs and stats on PHP usage growth since the original "Personal HomePage Tools" in 1995. Spoiler: it's up to 244m unique hostnames running PHP in some form, with possibly a higher host installbase, not to mention the hosts that hide the fact they're running PHP or hosts not surveyed by Netcraft's bots. Wow.

How to Make Async Requests in PHP
Calvin French-Owen, segment.io co-founder, shares his experiences trying to achieve what he terms asynchronous, but really he means non-blocking, HTTP client requests from PHP, from using immediately closed socket connections (with various techniques, and caveats), to batching requests with a second process, and forking out cURL processes. Some good ideas and conclusions.

Doctrine and SOLID
If the SOLID pattern is your thing then Benjamin Eberlei  gets down to the nitty gritty of getting your Doctrine2 models to use "value objects" which extract business logic components into child entities of the model. Benjamin continues to explain that unfortunately Doctrine doesn't support embedded objects like this, and until he manages to get support for them into core, he provides a few alternatives.

Benchmarking Cache Transcoders in PHP
Michael Nitschinger runs through a few techniques for serialising PHP objects, with an eye for storing objects in a caching system such as memcached. Some interesting benchmarks, with the wildcard being transforming objects to scalar values such as an array and serialising to JSON. The igbinary extension comes out on top for smaller sizes when serialising small sets of complex objects, but if speed is your thing the inbuilt object serialisation in PHP comes out on top.

Managing PHP Application Builds with Phing
Andy Podner covers a subject close to my heart, application build systems, with an old favourite Phing (I once created a Composer-like sandboxed system for Phing, called baskit) which is ant-like build system for PHP powered by verbose yet readable XML files. Andy goes over the first steps installing and setting up Phing for your project, as well as some future possibilities for what you can use it for.

Simply scale with Nginx, Memcached, PHP-FPM and APC
Although we recently linked to an article explaning that in a basic 1-to-1 comparison, with certain filesystem IO heavy ops disabled, Apache can keep up with Nginx on concurrent connections to PHP apps, James Fuller covers the many-many reasons for still choosing Nginx + PHP-FPM to deploy and scale your app.

7php - Zend Certifications Tips & Tricks
Khayrattee Wasseem interviews Zend Certified PHP Engineer Michelangelo van Dam about the Zend Certification itself, covering what it is, what it isn't, tips, tricks and addressing some widely held criticisms. Some very good advice for anyone thinking of becoming a ZCE (really, consider it, I did many years ago and it was quite worth it), and it's part 1 in a series of articles, so expect more helpful advice from other ZCEs in action.

How to Write Code That Embraces Change
This well written piece by Patkos Csaba is a good introduction to many object orientation techniques and patterns for keeping your codebase responsive to design changes while remaining maintainable. Patkos goes over a whole gamut of principles, from object cohesion, to orthogonality, dependency, coupling and even SOLID.

How to configure Symfony’s Service Container to use Twitter API
Gonzalo Ayuso shows us how to use Symfony2's wonderful service container system to wrap Twitter's REST API with just some simple YAML configuration, producing an elegant object mapping for the API.

Parallel PHPUnit
PHPUnit is great for running simple unit tests, and even integration tests, but as it runs sequentially in a single PHP process it doesn't exactly make use of all the available resources on most machines/VMs. Giorgio Sironi shares his experience trying out Paratest to run PHPUnit tests as multiple processes in parallel, producing aggregated results by parsing JUnit compatible XML output from the tests. Some good results, with a 25% saving in Giorgio's basic tests, but read carefully for caveats, and stay tuned for more results from his experiments.

Laravel 4: A Start at a RESTful API
Chris Fidao goes into a fair bit of detail on creating a simple RESTful HTTP API with L4, from initial project setup to routing, models and resourceful controllers.

Projects

ownCloud
Ever wanted your own Dropbox? ownCloud is an open source "cloud" (and by cloud we mean web service, effectively) storage, file syncing and media sharing platform, written in PHP (although other components, such as the syncing client, use different platforms).

MediaWiki Diff
Diff works either as an extension to MediaWiki (the software that powers Wikipedia) or as a standalone tool and provides utilities for generating diffs between JSON or JSON like data structures.

Temp-Token
A nice library for generating unique one-time time-limited tokens, especially good for password reset forms.

Flow
A fork of the original Twig templating system by Armin Ronacher, which you may know from the later fork that is part of Symfony. Flow extends the original concept of Twig with lots of extra features, influenced by Armin's later Python templating library Jinja2, the Symfony version of Twig and the Django templating system.

dnode-php
A PHP implementation of the DNode RPC protocol which allows you to write agents in PHP which can run and process jobs in a DNode cluster alongside Node.js agents, very cool.

woodhouse
A clever library for publishing build artefacts and status images to a GitHub repo from within a build process, e.g. updating build information in a README at the end of a build/test run on TravisCI for example.

 

RFCorner

Integrating Zend Optimizer+ into the PHP distribution
One of the biggest pieces of news this past week were these efforts to either release the Zend Optimizer+ system as the default opcode cache in PHP 5.5, or at the very least release it as an open source PECL extension if that falls through. Given the many advantages of O+ over the current "default" choice for opcode caches, APC, this is great news and I hope it becomes part of the default distro.
I was curious as to what Rasmus thought of it given that he is lead maintainer of APC, and his reply was promising.
PHP Magazin also have a good write up of the discussion, and the current main roadblock to the integration.
